# HTML 요소의 성격: 블록 vs 인라인

HTML 요소는 화면에 배치되는 방식에 따라 **블록 요소**와 **인라인 요소**로 나뉩니다.
이 차이를 이해하는 것이 웹 레이아웃 구성의 첫걸음입니다.

## 1. 블록 요소 (Block Elements)

새로운 줄에서 시작하며, 부모 요소의 **가로 너비 전체**를 차지하는 요소입니다.

### 비유: 블록 요소 = 가로로 꽉 찬 책상 서랍

> 서랍장을 생각해보세요.  
> 서랍 하나가 서랍장의 **가로 전체를 차지**하며, 서랍을 하나 열면 아래 서랍은 그 아래에 위치합니다.  
> 두 서랍이 나란히 같은 줄에 놓이지 않는 것처럼, 블록 요소도 항상 **새 줄에서 시작**합니다.

```
┌──────────────────────────────────┐
│          <h1> 제목               │  ← 줄 전체 차지
├──────────────────────────────────┤
│          <p> 문단                │  ← 줄 전체 차지
├──────────────────────────────────┤
│          <div> 구역              │  ← 줄 전체 차지
└──────────────────────────────────┘
```

### 주요 특징

- 항상 **새로운 줄**에서 시작합니다.
- 가로 너비(`width`)가 **부모의 100%** 를 차지합니다.
- `width`, `height`, `margin`, `padding` 모두 설정할 수 있습니다.
- 내부에 블록 요소와 인라인 요소를 모두 포함할 수 있습니다.

### 대표 태그

| 태그 | 설명 |
|------|------|
| `<h1>`~`<h6>` | 제목 (Heading) |
| `<p>` | 문단 (Paragraph) |
| `<div>` | 범용 블록 컨테이너 (Division) |
| `<ul>`, `<ol>`, `<li>` | 목록 |
| `<section>`, `<article>` | 시맨틱 구획 |

### 코드 예시

```html
<h1>Hello HTML! (h1)</h1>
<h2>Hello HTML! (h2)</h2>
<div>Hello HTML! (div)</div>
<p>Hello HTML! (p)</p>
```

위 4개 요소는 모두 새 줄에서 시작하며, 화면 가로 전체를 차지합니다.

```
┌────────────────────────────┐
│ Hello HTML! (h1)           │
├────────────────────────────┤
│ Hello HTML! (h2)           │
├────────────────────────────┤
│ Hello HTML! (div)          │
├────────────────────────────┤
│ Hello HTML! (p)            │
└────────────────────────────┘
```

## 2. 인라인 요소 (Inline Elements)

새로운 줄에서 시작하지 않으며, **텍스트(내용물) 너비만큼**만 공간을 차지하는 요소입니다.
줄 안(In-line)에 흐르듯 나란히 배치됩니다.

### 비유: 인라인 요소 = 문장 속 단어

> 글을 쓸 때 단어들이 왼쪽에서 오른쪽으로 이어지다가, 줄이 꽉 차면 다음 줄로 넘어가죠?  
> 인라인 요소도 마찬가지입니다. 요소들이 나란히 배치되다가 공간이 부족하면 자동으로 다음 줄로 넘어갑니다.

```
┌────────────────────────────────────────┐
│ [span1] [span2] [링크] 나란히 배치됨   │
└────────────────────────────────────────┘
```

### 주요 특징

- **줄 바꿈 없이** 다른 요소와 나란히 배치됩니다.
- `width`, `height`를 직접 지정할 수 없습니다. (내용 크기에 따라 자동 결정)
- 상하 `margin`은 적용되지 않습니다.
- ⚠️ **인라인 요소 안에 블록 요소를 넣을 수 없습니다.**

### 대표 태그

| 태그 | 설명 |
|------|------|
| `<span>` | 범용 인라인 컨테이너 |
| `<a>` | 하이퍼링크 (Anchor) |
| `<img>` | 이미지 |
| `<strong>`, `<em>` | 의미적 강조 |
| `<b>`, `<i>` | 시각적 강조 |

### 코드 예시

```html
<span>Hello HTML! (span1)</span>
<span>Hello HTML! (span2)</span>
<a href="#">Hello HTML! (link)</a>
```

위 3개 요소는 줄 바꿈 없이 나란히 배치되며, 텍스트 길이만큼만 너비를 차지합니다.

```
┌────────────────────────────────────────┐
│ Hello HTML! (span1) Hello HTML! (span2) Hello HTML! (link) │
└────────────────────────────────────────┘
```

## 3. 블록 vs 인라인 — 한눈에 비교

| 구분 | 블록 요소 | 인라인 요소 |
|------|-----------|-------------|
| 줄 바꿈 | 항상 새 줄에서 시작 | 줄 바꿈 없이 나란히 배치 |
| 너비 | 부모의 100% 차지 | 콘텐츠 크기만큼만 차지 |
| `width` / `height` | 지정 가능 | 지정 불가 |
| `margin` / `padding` | 상하좌우 모두 적용 | 좌우만 적용 (상하 무시) |
| 자식 포함 | 블록 + 인라인 모두 가능 | 인라인만 가능 |
| 대표 태그 | `div`, `p`, `h1`~`h6` | `span`, `a`, `img` |
| 용도 | 레이아웃 구역 나누기 | 텍스트 일부 강조·장식 |

### 실제 사용 사례로 이해하기

```
[뉴스 기사 페이지 구조]

┌───────────────────────────────┐
│  <h1>오늘의 뉴스</h1>         │  ← 블록: 줄 전체 차지
├───────────────────────────────┤
│  <p>                          │  ← 블록: 줄 전체 차지
│    오늘 서울에서               │
│    <strong>중요한 회의</strong>│  ← 인라인: 줄 안에서 굵게
│    가 열렸습니다.              │
│    <a href="#">자세히 보기</a> │  ← 인라인: 줄 안에서 링크
│  </p>                         │
└───────────────────────────────┘
```

> 블록 요소(`<p>`)가 문단 전체 영역을 잡고, 그 안에서 인라인 요소(`<strong>`, `<a>`)가 특정 텍스트만 꾸밉니다.

## 4. 핵심 태그 비교: `div` vs `span`

두 태그 모두 **자체적인 의미(Semantic)는 없지만**, 요소를 그룹화하고 스타일을 적용할 때 가장 많이 사용됩니다.

### `<div>` (Division) — 블록 컨테이너

문서의 섹션을 나누거나 여러 요소를 하나로 묶는 **블록 컨테이너**입니다.
레이아웃의 뼈대를 잡을 때 주로 사용합니다.

- 블록 요소와 인라인 요소를 모두 자식으로 포함할 수 있습니다.
- CSS `class`를 지정해 영역 단위로 스타일을 적용합니다.

```html
<div class="group-box">
  <h2>그룹 제목</h2>
  <p>div는 이렇게 <span>인라인 요소</span>와 블록 요소를 모두 묶을 수 있습니다.</p>
</div>
```

### `<span>` — 인라인 컨테이너

텍스트의 **특정 부분에만** 스타일을 입히거나 기능을 부여할 때 사용하는 **인라인 컨테이너**입니다.

- 문장 중간에 삽입되어 텍스트 흐름을 깨지 않습니다.
- 블록 요소를 자식으로 포함할 수 없습니다.

```html
<p>
  HTML 공부는 <span class="highlight">꾸준함</span>이 가장 중요합니다.<br>
  문단을 구성하는 <span class="underline">p 태그</span> 안에 사용되었습니다.
</p>
```

### div와 span은 언제 쓰나요?

| 상황 | 사용할 태그 | 이유 |
|------|-------------|------|
| 여러 요소를 하나의 영역으로 묶고 싶을 때 | `<div>` | 블록 컨테이너이므로 구역 분리에 적합 |
| 문장 중 일부 단어만 색상/크기를 바꾸고 싶을 때 | `<span>` | 인라인이므로 줄 흐름을 유지하면서 적용 |

### 의미 있는 구역에는 시맨틱 태그를 쓰세요

`<div>`는 의미가 없는 태그이므로, 역할이 명확한 구역에는 아래 태그들을 사용합니다.

| 상황 | `<div>` 대신 사용할 태그 |
|------|--------------------------|
| 페이지 상단 (로고, 메뉴) | `<header>` |
| 페이지 하단 (저작권, 연락처) | `<footer>` |
| 주요 콘텐츠 영역 | `<main>` |
| 독립적인 콘텐츠 (기사, 포스트) | `<article>` |
| 관련 콘텐츠 묶음 | `<section>` |
| 보조 정보 (사이드바) | `<aside>` |

## 5. 인라인 안에 블록을 넣으면 안 되는 이유

### 규칙: 인라인 요소는 인라인 요소만 포함할 수 있습니다

```html
<!-- ❌ 잘못된 예: <span>(인라인) 안에 <div>(블록)을 넣음 -->
<span>
  <div>이렇게 하면 안 됩니다</div>
</span>

<!-- ✅ 올바른 예: 블록 안에 인라인 -->
<div>
  <span>이건 괜찮습니다</span>
</div>
```

### 왜 안 되는 걸까요?

> 비유: 우편함 크기 규칙
>
> 인라인 요소는 **작은 우편함**이고, 블록 요소는 **큰 냉장고**입니다.  
> 작은 우편함 안에 냉장고를 넣을 수는 없죠.  
> 하지만 냉장고(블록) 안에 우편함(인라인)을 넣는 건 가능합니다.

브라우저마다 이를 처리하는 방식이 달라 **예상치 못한 레이아웃 깨짐**이 발생할 수 있습니다.
처음부터 올바른 중첩 규칙을 지키는 것이 가장 안전합니다.

### 올바른 중첩 원칙

| 포함 관계 | 허용 여부 |
|-----------|:---------:|
| 블록 안에 블록 | ✅ 가능 |
| 블록 안에 인라인 | ✅ 가능 |
| 인라인 안에 인라인 | ✅ 가능 |
| 인라인 안에 블록 | ❌ 불가 |

> 그리고 반드시 기억하세요: **인라인 요소 안에 블록 요소는 절대 넣지 마세요.**
