# Chapter 1 · 텍스트 구조화 태그

웹 페이지의 **가독성**과 **접근성**을 높이려면 제목과 본문을 명확히 구분하는 구조화가 필수입니다.  
올바른 구조화는 사용자뿐만 아니라 검색 엔진(SEO)과 스크린 리더(Screen Reader)가 페이지를 정확히 해석하는 데도 큰 영향을 미칩니다.

## 1. 제목 태그 `h1`~`h6`

주제나 섹션의 타이틀을 나타낼 때 사용합니다.  
`<h1>`부터 `<h6>`까지 총 **6단계**의 계층을 제공합니다.

> 비유: 책의 목차와 같습니다.  
> `<h1>`은 책 제목, `<h2>`는 대단원, `<h3>`는 소단원, `<h4>`~`<h6>`는 세부 항목에 해당합니다.

### 태그별 특징

| 태그 | 기본 크기 | 역할 |
|------|-----------|------|
| `<h1>` | 가장 큼 | 문서 전체의 대제목 (페이지당 1개 권장) |
| `<h2>` | 크게 | 주요 섹션의 제목 |
| `<h3>` | 중간 | 세부 섹션의 제목 |
| `<h4>` | 약간 작음 | 소제목 |
| `<h5>` | 작음 | 더 작은 소제목 |
| `<h6>` | 가장 작음 | 가장 낮은 단계의 제목 |

### 코드 예시

```html
<h1>h1: 가장 중요한 대제목</h1>
<h2>h2: 섹션의 주제</h2>
<h3>h3: 세부 섹션의 제목</h3>
<h4>h4: 소제목</h4>
<h5>h5: 더 작은 소제목</h5>
<h6>h6: 가장 낮은 단계의 제목</h6>
```

### SEO와 검색 엔진에 미치는 영향

구글 같은 검색 엔진은 웹 페이지를 분석할 때 `<h1>` 태그를 **"이 페이지의 핵심 주제"** 로 인식합니다.  
`<h2>`, `<h3>`는 세부 주제로 파악합니다.  
제목 태그를 올바른 계층 구조로 사용하면 **검색 결과 상위 노출**에 도움이 됩니다.

### 주의사항

- **`<h1>`은 페이지당 한 번만** — 문서의 핵심 주제이므로 단 한 번만 사용합니다.
- **계층 구조 준수** — `<h1>` 다음에 바로 `<h3>`로 건너뛰지 않습니다.
  ```
  ✅ 올바른 예: h1 → h2 → h3
  ❌ 잘못된 예: h1 → h3 (h2를 건너뜀)
  ```
- **디자인 목적 사용 금지** — 글씨를 크게 만들고 싶어서 `<h1>`을 남발하면 검색 엔진이 페이지의 주제를 잘못 파악합니다. 크기 조절은 CSS(`font-size`)로 해결합니다.

## 2. 문단 태그 `p`

본문의 문단(단락)을 구분할 때 사용하며, **Paragraph(문단)** 의 약자입니다.

> 비유: 글쓰기에서 새로운 생각이나 주제로 넘어갈 때 단락을 나누는 것과 같습니다.

### 특징

- 브라우저는 `<p>` 태그 앞뒤에 자동으로 **위·아래 여백(Margin)** 을 만들어 문단을 시각적으로 분리합니다.
- **블록 레벨 요소**로, 항상 새 줄에서 시작하고 화면 가로 전체를 차지합니다.
- 하나의 `<p>` 안에는 **하나의 주제·맥락**으로 묶인 텍스트만 넣습니다.

### 코드 예시

```html
<!-- 각 <p>는 독립된 문단으로 렌더링되며, 브라우저가 자동으로 여백을 추가합니다 -->
<p>첫 번째 문단입니다. 이 내용은 하나의 주제로 묶인 문장들입니다.</p>
<p>두 번째 문단입니다. 새로운 주제가 시작되면 새 p 태그를 사용합니다.</p>
```

### 왜 `<p>`를 써야 할까?

| 관점 | 효과 |
|------|------|
| **사용자** | 문단 간격이 생겨 글이 읽기 편해집니다 |
| **SEO** | 검색 엔진이 본문 텍스트를 단락 단위로 정확히 인식합니다 |
| **접근성** | 스크린 리더가 "단락 시작" "단락 끝"을 사용자에게 알려줍니다 |
| **유지보수** | CSS로 `p { line-height: 1.8; }` 처럼 문단 스타일을 일괄 관리할 수 있습니다 |

## 3. 줄 바꿈 태그 `br`

문단 **내에서** 강제로 줄을 바꿀 때 사용하며, **Line Break(줄 바꿈)** 의 약자입니다.

> 비유: 같은 문단이지만 Enter 키를 눌러 줄만 바꾸는 것과 같습니다.

### 특징

- 닫는 태그가 없는 **빈 태그(Void Element)** 입니다.
- 새로운 단락을 만드는 것이 아니라, 현재 줄만 끊고 다음 줄로 내려갑니다.
- 시(詩) 구절, 우편 주소처럼 **단락은 같지만 줄이 나뉘어야 하는** 경우에 적합합니다.

### 코드 예시

```html
<!-- 주소: 하나의 주소(하나의 의미)지만 줄이 나뉘어야 하는 경우 -->
<p>
  서울특별시 중구 세종대로 110<br>
  (태평로1가 31)
</p>

<!-- 시 구절: 행(行) 단위로 줄을 바꿔야 하는 경우 -->
<p>
  나 보기가 역겨워<br>
  가실 때에는<br>
  말없이 고이 보내 드리오리다.
</p>
```

### 주의사항

**`<br>`은 문단 구분에 사용하면 안 됩니다.**  
문단 간 간격을 넓히기 위해 `<br><br><br>` 처럼 여러 번 사용하는 것은 잘못된 방법입니다.

```html
<!-- ❌ 잘못된 예: 줄 바꿈을 여러 번 써서 간격을 만들려는 시도 -->
첫 번째 문단입니다.<br><br><br>
두 번째 문단입니다.

<!-- ✅ 올바른 예: p 태그로 문단을 나누고, 간격은 CSS로 조절 -->
<p>첫 번째 문단입니다.</p>
<p>두 번째 문단입니다.</p>
<!-- CSS: p { margin-bottom: 20px; } 로 간격 조절 -->
```

## 4. 글자 강조 태그

단순히 시각적으로 굵게/기울게 만드는 것과 **의미적으로 강조**하는 것은 다릅니다.  
이 차이가 중요한 이유는 스크린 리더가 의미 태그를 기준으로 음성을 조절하기 때문입니다.

### 굵게 표시하기

| 태그 | 이름 | 의미 | 스크린 리더 반응 |
|------|------|------|-----------------|
| `<b>` | Bold | 별다른 의미 없이 텍스트를 단순히 굵게 표시 | 일반적으로 강조하지 않음 |
| `<strong>` | Strong Importance | 내용이 **매우 중요함**을 나타냄 | 강조된 목소리로 읽음 |

```html
<p>이 제품의 가격은 <b>10,000원</b>입니다.</p>
<!-- b: 그냥 굵게 표시. "10,000원"에 특별한 중요성 없음 -->

<p><strong>경고: 이 약은 과다 복용 시 위험합니다.</strong></p>
<!-- strong: "경고" 내용이 매우 중요한 정보임을 명시 -->
```

### 기울여 표시하기

| 태그 | 이름 | 의미 | 주요 사용 사례 |
|------|------|------|---------------|
| `<i>` | Italic | 기술 용어, 외래어, 생각 등 스타일 목적 기울임 | 책 제목, 학명, 외국어 단어 |
| `<em>` | Emphasis | 문장에서 특정 단어를 의미적으로 강조 | "나는 그것을 **직접** 봤다" |

```html
<p>나는 <i>Homo sapiens</i>(인류) 입니다.</p>
<!-- i: 학명(라틴어) 표기에 관례적으로 기울임 사용 -->

<p>나는 그 사람이 <em>직접</em> 했다고 말했습니다.</p>
<!-- em: "직접"을 강조해서 읽어야 문장 의미가 달라짐 -->
```

### 시각적 태그 vs 의미적 태그 핵심 차이

| 구분 | 태그 | 브라우저 표시 | 의미 전달 | 스크린 리더 |
|------|------|:---:|:---:|:---:|
| 시각적 | `<b>`, `<i>` | 굵게/기울임 | ❌ 없음 | 일반 텍스트로 읽음 |
| 의미적 | `<strong>`, `<em>` | 굵게/기울임 | ✅ 있음 | 강조해서 읽음 |

> **정리:** 화면에서 보이는 결과는 같아도, 의미 있는 강조에는 `<strong>`·`<em>`을 사용해야 접근성과 SEO 모두 향상됩니다.

## 5. 인용 및 편집 태그

텍스트의 **출처**나 **변경 이력(추가/삭제)** 을 명시할 때 사용합니다.

### 인용 태그

| 태그 | 이름 | 용도 | 브라우저 기본 표시 |
|------|------|------|-------------------|
| `<q>` | Quote(인용) | 짧은 인용구 | 자동으로 `"따옴표"` 추가 |
| `<blockquote>` | Block Quote(블록 인용) | 긴 문단 단위 인용문 | 좌우 들여쓰기(여백) 적용 |

```html
<!-- q: 짧은 인용 — 브라우저가 따옴표를 자동으로 붙여줍니다 -->
<p>공자는 <q>배우고 때때로 익히면 기쁘지 아니한가</q>라고 했습니다.</p>

<!-- blockquote: 길고 독립적인 인용문 — cite 속성으로 출처 URL 명시 가능 -->
<blockquote cite="https://www.w3.org/">
  <p>The Web is for everyone.</p>
  <p>— Tim Berners-Lee (WWW 창시자)</p>
</blockquote>
```

### 편집 태그

문서의 **수정 이력**을 HTML로 표현할 때 사용합니다. 쇼핑몰 가격 할인 표시, 문서 변경 이력 등에 활용합니다.

| 태그 | 이름 | 시각적 표시 | 의미 |
|------|------|:-----------:|------|
| `<ins>` | Insert(삽입) | <u>밑줄</u> | 새롭게 추가된 텍스트 |
| `<del>` | Delete(삭제) | ~~취소선~~ | 삭제·수정된 이전 텍스트 |

```html
<!-- 쇼핑몰 할인 가격 표시 예시 -->
<p>
  정가: <del>50,000원</del>  ← 삭제된 기존 가격 (취소선)
  할인가: <ins>35,000원</ins>  ← 새로 적용된 가격 (밑줄)
</p>
```

## 6. 태그 비교 요약

| 태그 | 의미(Full Name) | 용도 | 레벨 | 빈 태그 |
|------|-----------------|------|:----:|:-------:|
| `<h1>`~`<h6>` | Heading | 계층적 제목 | 블록 | ✗ |
| `<p>` | Paragraph | 문단 구분 | 블록 | ✗ |
| `<br>` | Line Break | 문단 내 줄 바꿈 | 인라인 | ✓ |
| `<b>` | Bold | 시각적 굵게 (의미 없음) | 인라인 | ✗ |
| `<strong>` | Strong Importance | 의미적 강조 (굵게) | 인라인 | ✗ |
| `<i>` | Italic | 시각적 기울임 (의미 없음) | 인라인 | ✗ |
| `<em>` | Emphasis | 의미적 강조 (기울임) | 인라인 | ✗ |
| `<q>` | Quote | 짧은 인용구 (따옴표 자동) | 인라인 | ✗ |
| `<blockquote>` | Block Quote | 긴 인용문 (들여쓰기) | 블록 | ✗ |
| `<ins>` | Insert | 추가된 텍스트 (밑줄) | 인라인 | ✗ |
| `<del>` | Delete | 삭제된 텍스트 (취소선) | 인라인 | ✗ |

## 7. 실전 구조화 예제

아래는 제목, 문단, 줄 바꿈, 강조 태그를 함께 사용한 실제 문서 구조 예시입니다.

```html
<section>
  <h1>나의 학습 일지</h1>                 <!-- 문서 대제목 (1개만) -->

  <h2>오늘 배운 내용: HTML 텍스트 태그</h2>  <!-- 주요 섹션 제목 -->
  <p>
    오늘은 HTML의 텍스트 구조화 태그를 배웠습니다.<br>  <!-- 같은 문단 안에서 줄 바꿈 -->
    <strong>가장 중요한 점</strong>은 태그를 의미에 맞게 쓰는 것입니다.
  </p>

  <h3>주의할 점</h3>                      <!-- 세부 항목 제목 -->
  <p>
    <em>디자인을 위해</em> h 태그를 마음대로 골라 쓰면 안 됩니다.<br>
    구조에 맞는 태그 선택이 <strong>무엇보다 중요</strong>합니다.
  </p>

  <h3>참고 자료</h3>
  <p>공식 문서에 따르면 <q>HTML은 구조를 담당한다</q>고 명시되어 있습니다.</p>

  <blockquote>
    <p>Always code as if the guy who ends up maintaining your code is a violent psychopath who knows where you live.</p>
  </blockquote>
</section>
```

### 계층 구조 트리

```
section
├── h1   ← 문서 대제목 (페이지 전체에서 1개)
├── h2   ← 주요 섹션 제목
│   └── p  ← 본문 (내부에 br, strong, em 사용 가능)
├── h3   ← 세부 항목 제목
│   └── p
└── h3
    ├── p   (q 태그 포함)
    └── blockquote
```

---

> **핵심 원칙:** 태그는 **시각적 디자인이 아닌 의미와 구조**를 기준으로 선택하세요.  
> 스타일은 CSS로 제어하고, HTML은 데이터의 의미에 집중해야 합니다.  
> `<strong>`, `<em>` 같은 의미 태그는 스크린 리더에서도 적극 활용됩니다.
