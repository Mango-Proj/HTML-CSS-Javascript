# HTML 시맨틱 태그 (Semantic Tags)

시맨틱(Semantic)이란 **"의미가 있다"** 는 뜻입니다.
시맨틱 태그는 태그 이름만으로도 그 역할과 용도를 알 수 있는 HTML 요소입니다.

## 1. 시맨틱 vs 논 시맨틱

| 구분 | 시맨틱 태그 (Semantic) | 논 시맨틱 태그 (Non-semantic) |
|------|----------------------|------------------------------|
| **정의** | 태그 이름으로 의미와 용도를 알 수 있음 | 태그 이름만으로는 용도 파악 어려움 |
| **예시** | `<header>`, `<nav>`, `<main>`, `<article>` | `<div>`, `<span>` |
| **비유** | '거실', '안방', '화장실' 표지판 | '방1', '방2', '방3' 번호 표지판 |

```html
<!-- 논 시맨틱 방식 — 코드만 봐서는 각 div의 역할을 알 수 없음 -->
<div id="header"> ... </div>
<div id="main"> ... </div>
<div id="footer"> ... </div>

<!-- 시맨틱 방식 — 태그 이름만으로 역할이 명확히 드러남 -->
<header> ... </header>
<main> ... </main>
<footer> ... </footer>
```

## 2. 시맨틱 태그를 사용하는 이유

### SEO (검색 엔진 최적화)

검색 엔진(Google 등)이 페이지의 구조와 핵심 콘텐츠를 정확히 파악합니다.
`<main>` 안의 내용을 핵심 콘텐츠로, `<aside>`를 부가 정보로 인식해 검색 순위에 반영합니다.

### 웹 접근성 (Accessibility)

스크린 리더는 `<nav>`를 "내비게이션", `<main>`을 "주요 콘텐츠"로 사용자에게 음성으로 안내합니다.
시각 장애인도 페이지 구조를 정확하게 파악하고 원하는 영역으로 빠르게 이동할 수 있습니다.

### 유지보수 및 가독성

개발자가 코드를 볼 때 `<header>`, `<section>`, `<footer>`만으로도
어느 영역이 무슨 역할을 하는지 한눈에 파악할 수 있습니다.

## 3. 주요 시맨틱 태그

| 태그 | 역할 | 사용 위치 |
|------|------|-----------|
| `<header>` | 페이지/섹션 최상단 — 로고, 제목, 내비게이션 | `<body>` 또는 `<section>` 안 |
| `<nav>` | 내비게이션 — 메뉴 링크 모음 | 주로 `<header>` 안 |
| `<main>` | 문서의 핵심 본문 콘텐츠 | `<body>` 안, **페이지당 1개** |
| `<section>` | 주제별로 연관된 콘텐츠 구역 | `<main>` 안 |
| `<article>` | 독립적으로 배포·재사용 가능한 콘텐츠 | `<section>` 또는 `<main>` 안 |
| `<aside>` | 메인 콘텐츠와 간접 연관된 부가 정보 | `<main>` 옆 (사이드바) |
| `<footer>` | 페이지/섹션 최하단 — 저작권, 연락처 | `<body>` 또는 `<section>` 안 |

### 태그별 상세 설명

**`<header>`**
- 페이지 전체 또는 개별 섹션의 상단 영역입니다.
- 로고, 사이트명, `<nav>`, 검색창 등을 포함합니다.
- `position: sticky`로 스크롤 시 상단 고정하는 데 자주 사용됩니다.

**`<nav>`**
- 사이트의 주요 메뉴 링크 모음입니다.
- 스크린 리더가 이 영역을 "내비게이션"으로 인식해 건너뛸 수 있게 합니다.
- 페이지 내 앵커 링크(`href="#section-id"`)와 함께 사용하면 스크롤 이동이 가능합니다.

**`<main>`**
- 페이지의 핵심 본문 콘텐츠를 담습니다.
- ⚠️ **페이지당 단 하나만** 사용해야 합니다.
- `<header>`, `<footer>`, `<nav>`, `<aside>` 안에는 넣을 수 없습니다.

**`<section>`**
- 하나의 주제로 묶이는 콘텐츠 블록입니다.
- 반드시 `<h2>`~`<h6>` 제목 태그를 포함하는 것이 권장됩니다.
- `id` 속성을 부여하면 `<nav>`의 앵커 링크 대상이 됩니다.

**`<article>`**
- 블로그 포스트, 뉴스 기사, 댓글처럼 **단독으로도 의미를 가지는** 콘텐츠에 사용합니다.
- `<section>` 안에 포함되거나, 반대로 `<article>` 안에 `<section>`을 넣을 수도 있습니다.

**`<aside>`**
- 메인 콘텐츠와 직접적으로는 관련 없는 부가 정보 영역입니다.
- 광고, 사이드바, 관련 글, 추천 링크 등에 사용합니다.

**`<footer>`**
- 저작권 정보(`&copy;`), 연락처, 이용약관 링크 등을 담습니다.
- 검색 엔진은 `<footer>` 안의 콘텐츠를 핵심 내용이 아닌 부가 정보로 처리합니다.

## 4. 페이지 레이아웃 구조

```
┌─────────────────────────────────────────┐
│              <header>                   │
│   사이트 제목          <nav> 메뉴 링크   │
├─────────────────────────────────────────┤
│                 <main>                  │
│  ┌──────────────────────┐ ┌──────────┐  │
│  │      <section>       │ │ <aside>  │  │
│  │  ┌────────────────┐  │ │          │  │
│  │  │   <article>    │  │ │  사이드바 │  │
│  │  │  독립 콘텐츠   │  │ │  광고 등  │  │
│  │  └────────────────┘  │ │          │  │
│  └──────────────────────┘ └──────────┘  │
├─────────────────────────────────────────┤
│              <footer>                   │
│       저작권 / 연락처 / 이용약관         │
└─────────────────────────────────────────┘
```

### section vs article 선택 기준

| 질문 | 답 | 사용할 태그 |
|------|----|-----------:|
| 이 콘텐츠를 RSS 피드나 SNS에 단독으로 공유할 수 있나? | 예 | `<article>` |
| 이 콘텐츠는 같은 주제의 다른 콘텐츠와 함께 묶여야 하나? | 예 | `<section>` |
| 단순히 스타일 적용을 위해 묶는 것인가? | 예 | `<div>` |

## 5. 종합 예제

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>시맨틱 태그 예제</title>
</head>
<body>

  <!-- 페이지 상단: 제목 + 내비게이션 -->
  <header>
    <h1>코딩 학습 로드맵</h1>
    <nav>
      <ul>
        <!-- href="#html" → <section id="html">로 스크롤 이동 -->
        <li><a href="#html">HTML</a></li>
        <li><a href="#css">CSS</a></li>
        <li><a href="#js">JavaScript</a></li>
      </ul>
    </nav>
  </header>

  <!-- 핵심 본문: 페이지당 단 한 번만 사용 -->
  <main>

    <!-- 주제별 콘텐츠 구역: id로 nav 앵커 링크 연결 -->
    <section id="html">
      <h2>HTML 기초</h2>
      <p>웹 페이지의 뼈대를 만드는 언어입니다.</p>

      <!-- 독립적 콘텐츠: 단독으로 배포·재사용 가능한 블록 -->
      <article>
        <h3>시맨틱 태그란 무엇인가?</h3>
        <p>태그 자체가 의미를 가지는 요소를 말합니다...</p>
      </article>
    </section>

    <!-- 부가 정보: 메인 흐름과 간접 연관된 사이드바 -->
    <aside>
      <h4>관련 광고</h4>
      <p>지금 바로 코딩 강의를 신청하세요!</p>
    </aside>

  </main>

  <!-- 페이지 하단: 저작권, 연락처, 이용약관 -->
  <footer>
    <!-- &copy; : 저작권 기호(©) HTML 특수문자 엔티티 -->
    <p>&copy; 2026 Coding Valley. All rights reserved.</p>
    <p>연락처: info@example.com | <a href="#">이용약관</a></p>
  </footer>

</body>
</html>
```

|------|-----------|
| `example.html` | 시맨틱 태그 7종을 활용한 표준 웹 페이지 전체 구조 |
