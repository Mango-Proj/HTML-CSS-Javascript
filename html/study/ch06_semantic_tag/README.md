# HTML 시맨틱 태그 (Semantic Tags)

시맨틱(Semantic)이란 **"의미가 있다"** 는 뜻입니다.
시맨틱 태그는 태그 이름만으로도 그 역할과 용도를 알 수 있는 HTML 요소입니다.

## 1. 시맨틱 vs 논 시맨틱

### 비유: 방에 표지판이 있는 집 vs 번호만 있는 집

> 집에 방이 4개 있다고 상상해보세요.  
>
> **논 시맨틱 방식** — 방마다 "방1", "방2", "방3", "방4" 라는 번호만 붙어 있습니다.  
> 들어가봐야 거실인지, 화장실인지, 주방인지 알 수 있습니다.
>
> **시맨틱 방식** — "거실", "주방", "침실", "화장실" 이라고 표지판이 붙어 있습니다.  
> 문을 열기 전부터 어떤 방인지 알 수 있습니다.

HTML에서도 마찬가지입니다:

| 구분 | 시맨틱 태그 (Semantic) | 논 시맨틱 태그 (Non-semantic) |
|------|----------------------|------------------------------|
| **정의** | 태그 이름으로 의미와 용도를 알 수 있음 | 태그 이름만으로는 용도 파악 어려움 |
| **예시** | `<header>`, `<nav>`, `<main>`, `<article>` | `<div>`, `<span>` |
| **비유** | '거실', '주방', '침실' 표지판 | '방1', '방2', '방3' 번호 표지판 |

```html
<!-- 논 시맨틱 방식 — 코드만 봐서는 각 div의 역할을 알 수 없음 -->
<div id="header"> ... </div>
<div id="main"> ... </div>
<div id="footer"> ... </div>

<!-- 시맨틱 방식 — 태그 이름만으로 역할이 명확히 드러남 -->
<header> ... </header>
<main> ... </main>
<footer> ... </footer>
```

두 코드는 화면에 똑같이 보이지만, 브라우저·검색 엔진·스크린 리더가 이해하는 방식이 완전히 다릅니다.

## 2. 시맨틱 태그를 사용하는 이유

### SEO (검색 엔진 최적화) — 검색 순위와 직결됩니다

검색 엔진(구글, 네이버 등)은 페이지를 자동으로 읽고 분석합니다.
시맨틱 태그를 사용하면 검색 엔진이 콘텐츠의 구조와 핵심 내용을 정확히 파악합니다.

| 상황 | 설명 |
|------|------|
| `<main>` 안의 내용 | "이 페이지의 핵심 콘텐츠"로 인식 → 검색 결과에 적극 반영 |
| `<nav>` 안의 링크 | "사이트 내부 구조 파악"에 활용 |
| `<aside>` 안의 내용 | "부가 정보"로 인식 → 핵심 내용보다 낮은 우선순위 |
| `<footer>` 안의 내용 | "저작권·연락처 등 부가 정보"로 처리 |

> **실제 효과:** 같은 내용이라도 시맨틱 태그를 올바르게 사용한 페이지가 검색 결과 상위에 노출될 가능성이 높습니다.

---

### 웹 접근성 (Accessibility) — 모든 사람이 사용할 수 있는 웹

스크린 리더(화면 낭독 소프트웨어)는 시각 장애인을 위해 화면 내용을 음성으로 읽어줍니다.
시맨틱 태그가 있으면 단순히 텍스트를 읽는 것을 넘어 **구역의 역할**까지 안내해줍니다.

| 태그 | 스크린 리더가 읽어주는 내용 |
|------|---------------------------|
| `<nav>` | "내비게이션 영역입니다. 메뉴 항목: 홈, 소개, 연락처..." |
| `<main>` | "주요 콘텐츠 영역입니다." |
| `<aside>` | "보조 정보 영역입니다." |
| `<footer>` | "페이지 하단 영역입니다." |

시각 장애인은 이 안내를 듣고 **원하는 영역으로 바로 건너뛸 수 있습니다.**
`<div id="header">` 만으로는 이런 안내가 제공되지 않습니다.

---

### 유지보수 및 협업 — 코드를 읽기 쉬워집니다

```html
<!-- 6개월 후 이 코드를 다시 봤을 때 -->

<!-- ❌ 논 시맨틱 방식: 각 div가 무슨 역할인지 파악하기 어려움 -->
<div class="wrap">
  <div class="top"> ... </div>
  <div class="mid">
    <div class="left"> ... </div>
    <div class="right"> ... </div>
  </div>
  <div class="bot"> ... </div>
</div>

<!-- ✅ 시맨틱 방식: 읽는 순간 구조가 바로 이해됨 -->
<body>
  <header> ... </header>
  <main>
    <section> ... </section>
    <aside> ... </aside>
  </main>
  <footer> ... </footer>
</body>
```

## 3. 주요 시맨틱 태그

| 태그 | 역할 | 사용 위치 |
|------|------|-----------|
| `<header>` | 페이지/섹션 최상단 — 로고, 제목, 내비게이션 | `<body>` 또는 `<section>` 안 |
| `<nav>` | 내비게이션 — 메뉴 링크 모음 | 주로 `<header>` 안 |
| `<main>` | 문서의 핵심 본문 콘텐츠 | `<body>` 안, **페이지당 1개** |
| `<section>` | 주제별로 연관된 콘텐츠 구역 | `<main>` 안 |
| `<article>` | 독립적으로 배포·재사용 가능한 콘텐츠 | `<section>` 또는 `<main>` 안 |
| `<aside>` | 메인 콘텐츠와 간접 연관된 부가 정보 | `<main>` 옆 (사이드바) |
| `<footer>` | 페이지/섹션 최하단 — 저작권, 연락처 | `<body>` 또는 `<section>` 안 |

### 태그별 상세 설명

**`<header>`**
- 페이지 전체 또는 개별 섹션의 상단 영역입니다.
- 로고, 사이트명, `<nav>`, 검색창 등을 포함합니다.
- CSS `position: sticky`로 스크롤 시 상단 고정하는 데 자주 사용됩니다.

**`<nav>`**
- 사이트의 주요 메뉴 링크 모음입니다.
- 스크린 리더가 이 영역을 "내비게이션"으로 인식해 건너뛸 수 있게 합니다.
- 페이지 내 앵커 링크(`href="#section-id"`)와 함께 사용하면 해당 섹션으로 스크롤 이동이 가능합니다.

**`<main>`**
- 페이지의 핵심 본문 콘텐츠를 담습니다.
- ⚠️ **페이지당 단 하나만** 사용해야 합니다.
- `<header>`, `<footer>`, `<nav>`, `<aside>` 안에는 넣을 수 없습니다.

**`<section>`**
- 하나의 주제로 묶이는 콘텐츠 블록입니다.
- `<h2>`~`<h6>` 제목 태그를 포함하는 것이 권장됩니다.
- `id` 속성을 부여하면 `<nav>`의 앵커 링크 대상이 됩니다.

**`<article>`**
- 블로그 포스트, 뉴스 기사, 댓글처럼 **단독으로도 의미를 가지는** 콘텐츠에 사용합니다.
- `<section>` 안에 포함되거나, 반대로 `<article>` 안에 `<section>`을 넣을 수도 있습니다.

**`<aside>`**
- 메인 콘텐츠와 직접적으로는 관련 없는 부가 정보 영역입니다.
- 광고, 사이드바, 관련 글, 추천 링크 등에 사용합니다.

**`<footer>`**
- 저작권 정보(`&copy;`), 연락처, 이용약관 링크 등을 담습니다.
- 검색 엔진은 `<footer>` 안의 콘텐츠를 핵심 내용이 아닌 부가 정보로 처리합니다.

## 4. 페이지 레이아웃 구조

실제 웹 페이지는 아래와 같은 구조로 시맨틱 태그를 조합합니다.

```
┌─────────────────────────────────────────┐
│              <header>                   │
│   사이트 제목       <nav> 메뉴 링크     │
├─────────────────────────────────────────┤
│                 <main>                  │
│  ┌──────────────────────┐ ┌──────────┐  │
│  │      <section>       │ │ <aside>  │  │
│  │                      │ │          │  │
│  │  ┌────────────────┐  │ │  관련 글  │  │
│  │  │   <article>    │  │ │  광고 등  │  │
│  │  │  독립 콘텐츠   │  │ │          │  │
│  │  └────────────────┘  │ └──────────┘  │
│  └──────────────────────┘               │
├─────────────────────────────────────────┤
│              <footer>                   │
│       저작권 / 연락처 / 이용약관         │
└─────────────────────────────────────────┘
```

## 5. section vs article 구분법

`<section>`과 `<article>`은 둘 다 콘텐츠 구역을 나타내지만 쓰임새가 다릅니다.

### 핵심 질문으로 구분하기

> **"이 콘텐츠를 잘라내서 다른 사이트에 그대로 옮겨도 의미가 통하나요?"**
>
> - **예** → `<article>` (독립적인 콘텐츠)
> - **아니오** → `<section>` (맥락이 있어야 의미가 통하는 구역)

### 실생활 예시로 이해하기

| 콘텐츠 예시 | 판단 | 사용할 태그 |
|------------|------|-------------|
| 블로그 글 "오늘 배운 HTML 정리" | 다른 사이트에 퍼가도 독립적으로 의미 있음 | `<article>` |
| 쇼핑몰의 "이번 주 추천 상품" 구역 | 해당 사이트의 맥락이 있어야 의미 있음 | `<section>` |
| 뉴스 기사 "오늘의 날씨 예보" | RSS 피드로 단독 배포 가능 | `<article>` |
| FAQ 페이지의 "배송 관련 질문" 묶음 | 페이지의 일부 구역, 단독으로는 의미 약함 | `<section>` |
| 유튜브 댓글 하나 | 단독으로도 의미 있는 독립 콘텐츠 | `<article>` |
| 단순히 CSS 스타일을 묶기 위한 구역 | 의미 없이 그루핑만 하는 경우 | `<div>` |

### 중첩 구조도 가능합니다

```html
<!-- 뉴스 사이트 예시 -->
<section id="sports-news">         <!-- 스포츠 뉴스 구역 -->
  <h2>스포츠</h2>

  <article>                        <!-- 기사 1: 단독으로 의미 있음 -->
    <h3>월드컵 예선 결과</h3>
    <p>한국 대표팀이 ...</p>
  </article>

  <article>                        <!-- 기사 2: 단독으로 의미 있음 -->
    <h3>야구 리그 순위</h3>
    <p>이번 주 순위는 ...</p>
  </article>
</section>
```

## 6. 종합 예제

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>시맨틱 태그 예제</title>
</head>
<body>

  <!-- 페이지 상단: 로고, 사이트명, 메뉴 -->
  <header>
    <h1>코딩 학습 로드맵</h1>
    <nav>
      <ul>
        <!-- href="#html" → 아래의 <section id="html">로 스크롤 이동 -->
        <li><a href="#html">HTML</a></li>
        <li><a href="#css">CSS</a></li>
        <li><a href="#js">JavaScript</a></li>
      </ul>
    </nav>
  </header>

  <!-- 핵심 본문: 페이지당 단 한 번만 사용 -->
  <main>

    <!-- 주제별 콘텐츠 구역 (id로 nav 앵커 링크와 연결) -->
    <section id="html">
      <h2>HTML 기초</h2>
      <p>웹 페이지의 뼈대를 만드는 언어입니다.</p>

      <!--
        article: 이 기사만 따로 잘라내서 RSS나 다른 사이트에 게시해도
        독립적으로 의미가 통하는 콘텐츠
      -->
      <article>
        <h3>시맨틱 태그란 무엇인가?</h3>
        <p>태그 자체가 의미를 가지는 요소를 말합니다...</p>
      </article>

      <article>
        <h3>블록 vs 인라인 요소</h3>
        <p>HTML 요소의 배치 방식에 따른 두 가지 분류...</p>
      </article>
    </section>

    <section id="css">
      <h2>CSS 기초</h2>
      <p>웹 페이지의 스타일을 담당하는 언어입니다.</p>
    </section>

    <!-- aside: 메인 내용과 간접 연관된 부가 정보 (사이드바) -->
    <aside>
      <h4>관련 강의 추천</h4>
      <p>지금 바로 무료 코딩 강의를 시작해보세요!</p>
    </aside>

  </main>

  <!-- 페이지 하단: 저작권, 연락처, 이용약관 -->
  <footer>
    <!-- &copy; : 저작권 기호(©) HTML 특수문자 엔티티 -->
    <p>&copy; 2026 Coding Valley. All rights reserved.</p>
    <p>연락처: info@example.com | <a href="#">이용약관</a></p>
  </footer>

</body>
</html>
```
