# HTML 기초 학습 가이드

HTML의 핵심 태그를 6개 챕터로 나누어 단계별로 학습합니다.

---

## HTML이란 무엇인가?

**HTML(HyperText Markup Language)** 은 웹 페이지의 **구조와 내용을 정의하는 언어**입니다.

> 비유: 건물을 지을 때 가장 먼저 **설계도**를 그리듯,  
> HTML은 웹 페이지의 뼈대(골조)를 만드는 역할을 합니다.  
> 인테리어(스타일)는 CSS가, 자동문·엘리베이터(동작)는 JavaScript가 담당합니다.

### HTML, CSS, JavaScript의 역할 분담

| 언어 | 역할 | 비유 |
|------|------|------|
| **HTML** | 내용과 구조 — 제목, 문단, 이미지, 링크 배치 | 건물의 **뼈대와 방 구조** |
| **CSS** | 시각적 스타일 — 색상, 글꼴, 배치 | 건물의 **인테리어와 외장** |
| **JavaScript** | 동적 동작 — 버튼 클릭, 데이터 불러오기 | 건물의 **엘리베이터, 자동문** |

### 브라우저는 HTML을 어떻게 처리하나?

브라우저(Chrome, Safari 등)는 HTML 파일을 받으면 코드를 위에서 아래로 **한 줄씩 읽고 해석**합니다.  
이 과정을 **파싱(Parsing)** 이라고 하며, 파싱이 끝나면 화면에 웹 페이지가 표시됩니다.

```html
<!-- 브라우저는 이 코드를 읽어 화면에 표시합니다 -->
<h1>안녕하세요</h1>   → 크고 굵은 제목으로 표시
<p>반갑습니다.</p>     → 일반 문단으로 표시
```

## 알아두면 좋은 공통 개념

#### SEO (검색 엔진 최적화)란?

**SEO(Search Engine Optimization)** 는 구글, 네이버 같은 검색 엔진이 내 페이지를 잘 이해하고  
검색 결과 상위에 노출되도록 최적화하는 것입니다.  
HTML 태그를 올바르게 사용하면 검색 엔진이 페이지 내용을 더 정확히 파악해 검색 순위가 향상됩니다.

#### 접근성(Accessibility)과 스크린 리더란?

**접근성**은 시각 장애인, 청각 장애인 등 다양한 사용자가 웹을 이용할 수 있도록 하는 것입니다.  
**스크린 리더(Screen Reader)** 는 시각 장애인이 사용하는 소프트웨어로,  
화면의 텍스트와 HTML 구조를 음성으로 읽어줍니다.  
`alt`, `<label>`, `<strong>` 같은 태그가 올바르게 사용될수록 스크린 리더가 내용을 정확히 전달합니다.

#### 빈 태그(Void Element)란?

대부분의 HTML 태그는 `<시작>내용</끝>` 형태입니다.  
그런데 `<img>`, `<br>`, `<input>` 처럼 **내용을 담지 않고 닫는 태그가 없는** 태그를 **빈 태그(Void Element)** 라고 합니다.

## Chapter 1 — 텍스트 구조화 태그

> 📁 [`ch01_text_tag/README.md`](./ch01_text_tag/README.md)

**학습 목표:** 텍스트에 의미와 구조를 부여하는 태그를 구분하여 사용합니다.

### 주요 태그

| 태그 | 역할 |
|------|------|
| `<h1>`~`<h6>` | 6단계 계층적 제목 — `<h1>`은 페이지당 1개 권장 |
| `<p>` | 문단(Paragraph) — 블록 요소, 자동 상하 여백 |
| `<br>` | 문단 내 줄 바꿈 — 빈 태그(종료 태그 없음) |
| `<strong>` / `<b>` | 굵게 — `<strong>`은 의미적 강조, `<b>`는 시각적 표현 |
| `<em>` / `<i>` | 기울임 — `<em>`은 의미적 강조, `<i>`는 시각적 표현 |
| `<q>` | 짧은 인라인 인용구 — 브라우저가 자동으로 따옴표 추가 |
| `<blockquote>` | 긴 블록 단위 인용문 — 들여쓰기 자동 적용 |
| `<ins>` / `<del>` | 추가된 텍스트(밑줄) / 삭제된 텍스트(취소선) |

### 핵심 원칙

- 태그는 **시각적 디자인이 아닌 의미**를 기준으로 선택합니다.
- 크기·스타일 조절은 CSS로, HTML은 구조와 의미에만 집중합니다.
- `<h1>` → `<h2>` → `<h3>` 순서를 지키고 단계를 건너뛰지 않습니다.
- 문단 간격 조절에 `<br>`을 여러 번 사용하지 않고 `<p>` + CSS `margin`을 사용합니다.

## Chapter 2 — 미디어 & 링크

> 📁 [`ch02_media_tag/README.md`](./ch02_media_tag/README.md)

**학습 목표:** 이미지, 오디오, 비디오, 하이퍼링크를 웹 페이지에 올바르게 삽입합니다.

### 주요 태그

| 태그 | 역할 | 필수 속성 |
|------|------|-----------|
| `<img>` | 이미지 삽입 — 빈 태그 | `src`, `alt` |
| `<figure>` / `<figcaption>` | 이미지 + 캡션 시맨틱 묶음 | — |
| `<audio>` | 오디오 재생 플레이어 | `src` 또는 `<source>` |
| `<video>` | 비디오 재생 플레이어 | `src` 또는 `<source>` |
| `<source>` | 여러 미디어 포맷 지정 | `src`, `type` |
| `<a>` | 하이퍼링크 | `href` |

### 핵심 원칙

- `<img>`에는 **반드시 `alt` 속성**을 작성합니다. (접근성 + SEO)
- `<audio>` / `<video>`에 `controls` 속성이 없으면 재생 버튼이 표시되지 않습니다.
- 여러 포맷을 지원하려면 `<source>` 태그를 중첩하여 폴백(fallback)을 제공합니다.

## Chapter 3 — 블록 & 인라인 요소

> 📁 [`ch03_block_inline_tag/README.md`](./ch03_block_inline_tag/README.md)

**학습 목표:** 블록 요소와 인라인 요소의 차이를 이해하고 올바르게 중첩합니다.

### 핵심 개념

| 구분 | 특징 | 대표 태그 |
|------|------|-----------|
| **블록(Block)** | 새 줄에서 시작, 가로 전체 차지 | `<div>`, `<p>`, `<h1>`~`<h6>` |
| **인라인(Inline)** | 텍스트 흐름 내에 위치, 내용 너비만큼 차지 | `<span>`, `<a>`, `<strong>`, `<em>` |

### 핵심 원칙

- 인라인 요소 안에 블록 요소를 넣으면 안 됩니다. (`<span>` 안에 `<div>` 불가)
- 의미 있는 구역에는 `<div>` 대신 시맨틱 태그를 사용합니다.

## Chapter 4 — 목록 & 표

> 📁 [`ch04_list_table_tag/README.md`](./ch04_list_table_tag/README.md)

**학습 목표:** 항목 나열에는 목록 태그를, 2차원 데이터에는 표 태그를 사용합니다.

### 목록 태그

| 태그 | 이름 | 마커 | 사용 예시 |
|------|------|:----:|-----------|
| `<ol>` | Ordered List | 1, 2, 3... | 순위, 단계별 절차 |
| `<ul>` | Unordered List | ● | 메뉴, 기능 목록 |
| `<dl>` | Description List | 없음 | 용어 사전, FAQ |

### 핵심 원칙

- `<ul>` / `<ol>`의 직계 자식은 반드시 `<li>`여야 합니다.
- 표(`<table>`)는 **데이터 표현 전용**입니다. 레이아웃 목적으로 사용하지 않습니다.

## Chapter 5 — 폼 & 입력 요소

> 📁 [`ch05_form_tag/README.md`](./ch05_form_tag/README.md)

**학습 목표:** 사용자 입력을 받아 서버로 전달하는 폼(Form) 요소를 올바르게 구성합니다.

### 핵심 원칙

- `name` 속성이 없으면 서버에서 데이터를 식별할 수 없습니다. (**필수**)
- 비밀번호·개인정보는 반드시 `method="post"`를 사용합니다.
- `<label for="id">`와 `<input id="id">`를 연결하면 라벨 클릭 시 포커스가 이동합니다.

## Chapter 6 — 시맨틱 태그

> 📁 [`ch06_semantic_tag/README.md`](./ch06_semantic_tag/README.md)

**학습 목표:** 태그 이름만으로 역할을 알 수 있는 시맨틱 태그로 페이지 구조를 설계합니다.

### 주요 시맨틱 태그

| 태그 | 역할 |
|------|------|
| `<header>` | 페이지 상단 — 로고, 제목, 내비게이션 |
| `<nav>` | 내비게이션 메뉴 |
| `<main>` | 핵심 본문 (페이지당 1개) |
| `<section>` | 주제별 콘텐츠 구역 |
| `<article>` | 독립적으로 배포 가능한 콘텐츠 |
| `<aside>` | 부가 정보 (사이드바) |
| `<footer>` | 페이지 하단 — 저작권, 연락처 |

## 태그 전체 요약표

| 챕터 | 핵심 태그 | 핵심 개념 |
|------|-----------|-----------|
| 1. 텍스트 | `h1~h6`, `p`, `br`, `strong`, `em`, `q`, `blockquote`, `ins`, `del` | 의미 vs 시각적 표현 구분 |
| 2. 미디어/링크 | `img`, `audio`, `video`, `source`, `a`, `figure` | alt 속성, 다양한 href 유형 |
| 3. 블록/인라인 | `div`, `span` | 레이아웃 레벨 차이, 올바른 중첩 |
| 4. 목록/표 | `ul`, `ol`, `dl`, `table`, `thead`, `tbody`, `th`, `td` | 데이터 구조에 맞는 태그 선택 |
| 5. 폼 | `form`, `input`, `label`, `select`, `textarea`, `button` | name/id 역할 구분, GET vs POST |
| 6. 시맨틱 | `header`, `nav`, `main`, `section`, `article`, `aside`, `footer` | 의미 있는 구조 설계 |

---

> **공통 핵심 원칙:** 태그는 **시각적 결과가 아닌 데이터의 의미**를 기준으로 선택하세요.  
> 스타일은 CSS로 제어하고, HTML은 구조와 의미에 집중합니다.  
> 시맨틱한 HTML은 SEO, 접근성, 유지보수 모두를 향상시킵니다.
