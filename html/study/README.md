## Chapter 1 — 텍스트 구조화 태그

> 📁 [`text_tag/README.md`](./text_tag/README.md)

**학습 목표:** 텍스트에 의미와 구조를 부여하는 태그를 구분하여 사용합니다.

### 주요 태그

| 태그 | 역할 |
|------|------|
| `<h1>`~`<h6>` | 6단계 계층적 제목 — `<h1>`은 페이지당 1개 권장 |
| `<p>` | 문단(Paragraph) — 블록 요소, 자동 상하 여백 |
| `<br>` | 문단 내 줄 바꿈 — 빈 태그(종료 태그 없음) |
| `<strong>` / `<b>` | 굵게 — `<strong>`은 의미적 강조, `<b>`는 시각적 표현 |
| `<em>` / `<i>` | 기울임 — `<em>`은 의미적 강조, `<i>`는 시각적 표현 |
| `<q>` | 짧은 인라인 인용구 — 브라우저가 자동으로 따옴표 추가 |
| `<blockquote>` | 긴 블록 단위 인용문 — 들여쓰기 자동 적용 |
| `<ins>` / `<del>` | 추가된 텍스트(밑줄) / 삭제된 텍스트(취소선) |

### 핵심 원칙

- 태그는 **시각적 디자인이 아닌 의미**를 기준으로 선택합니다.
- 크기·스타일 조절은 CSS로, HTML은 구조와 의미에만 집중합니다.
- `<h1>` → `<h2>` → `<h3>` 순서를 지키고 단계를 건너뛰지 않습니다.
- 문단 간격 조절에 `<br>`을 여러 번 사용하지 않고 `<p>` + CSS `margin`을 사용합니다.

### 실습 파일

| 파일 | 학습 내용 |
|------|-----------|
| `title.html` | h1~h6 계층 구조 시각화 |
| `paragraph.html` | `<p>` 태그와 문단 구분 |
| `linebreak.html` | `<br>` 태그 — 주소, 시 구절 |
| `emphasis.html` | b vs strong, i vs em 비교 |
| `quote_edit.html` | q, blockquote, ins, del |
| `self_introdution.html` | 텍스트 태그 종합 실습 |
| `structure_example.html` | 계층 구조 종합 예제 |

## Chapter 2 — 블록 & 인라인 요소

> 📁 [`block_inline_tag/README.md`](./block_inline_tag/README.md)

**학습 목표:** 블록 요소와 인라인 요소의 차이를 이해하고 올바르게 중첩합니다.

### 핵심 개념

| 구분 | 특징 | 대표 태그 |
|------|------|-----------|
| **블록(Block)** | 새 줄에서 시작, 가로 전체 차지 | `<div>`, `<p>`, `<h1>`~`<h6>` |
| **인라인(Inline)** | 텍스트 흐름 내에 위치, 내용 너비만큼 차지 | `<span>`, `<a>`, `<strong>`, `<em>` |

### div vs span

| 태그 | 레벨 | 용도 |
|------|------|------|
| `<div>` | 블록 | 여러 요소를 하나의 그룹으로 묶는 레이아웃 컨테이너 |
| `<span>` | 인라인 | 텍스트 일부에 스타일을 입히는 인라인 컨테이너 |

### 핵심 원칙

- 인라인 요소 안에 블록 요소를 넣으면 안 됩니다. (`<span>` 안에 `<div>` 불가)
- 의미 있는 구역에는 `<div>` 대신 시맨틱 태그(`<header>`, `<main>`, `<section>` 등)를 사용합니다.
- `display: block` / `display: inline` CSS 속성으로 성질을 변경할 수 있습니다.

### 실습 파일

| 파일 | 학습 내용 |
|------|-----------|
| `block.html` | 블록 요소의 레이아웃 특성 |
| `inline.html` | 인라인 요소의 텍스트 흐름 특성 |
| `div_span.html` | div, span 컨테이너 활용 |

## Chapter 3 — 목록 & 표

> 📁 [`list_table_tag/README.md`](./list_table_tag/README.md)

**학습 목표:** 항목 나열에는 목록 태그를, 2차원 데이터에는 표 태그를 사용합니다.

### 목록 태그

| 태그 | 이름 | 마커 | 사용 예시 |
|------|------|:----:|-----------|
| `<ol>` | Ordered List | 1, 2, 3... | 순위, 단계별 절차 |
| `<ul>` | Unordered List | ● | 메뉴, 기능 목록 |
| `<dl>` | Description List | 없음 | 용어 사전, FAQ |
| `<li>` | List Item | — | ol / ul 안의 각 항목 |
| `<dt>` / `<dd>` | Term / Detail | — | dl 안의 용어와 설명 |

### 표 태그

| 태그 | 역할 |
|------|------|
| `<table>` | 표 전체 컨테이너 |
| `<caption>` | 표 제목 (접근성 필수) |
| `<thead>` / `<tbody>` / `<tfoot>` | 머리글 / 본문 / 바닥글 구역 |
| `<tr>` | 가로 행 |
| `<th>` / `<td>` | 제목 셀 (굵게+가운데) / 데이터 셀 |
| `colspan` / `rowspan` | 열 병합 / 행 병합 속성 |

### 핵심 원칙

- `<ul>` / `<ol>`의 직계 자식은 반드시 `<li>`여야 합니다.
- 표(`<table>`)는 **데이터 표현 전용**입니다. 레이아웃 목적으로 사용하지 않습니다.
- `border="1"` 같은 HTML 속성 방식 대신 CSS `border-collapse`로 처리합니다.

### 실습 파일

| 파일 | 학습 내용 |
|------|-----------|
| `list.html` | ul / ol 및 중첩 목록 구조 |
| `table.html` | table 구성 태그 및 colspan 셀 병합 |

## Chapter 4 — 미디어 & 링크

> 📁 [`media_tag/README.md`](./media_tag/README.md)

**학습 목표:** 이미지, 오디오, 비디오, 하이퍼링크를 웹 페이지에 올바르게 삽입합니다.

### 주요 태그

| 태그 | 역할 | 필수 속성 |
|------|------|-----------|
| `<img>` | 이미지 삽입 — 빈 태그 | `src`, `alt` |
| `<figure>` / `<figcaption>` | 이미지 + 캡션 시맨틱 묶음 | — |
| `<audio>` | 오디오 재생 플레이어 | `src` 또는 `<source>` |
| `<video>` | 비디오 재생 플레이어 | `src` 또는 `<source>` |
| `<source>` | 여러 미디어 포맷 지정 | `src`, `type` |
| `<a>` | 하이퍼링크 | `href` |

### 링크 유형 (`<a href="...">`)

| 값 | 기능 |
|----|------|
| `https://...` | 외부 웹사이트 이동 |
| `./파일.html` | 내부 페이지 이동 |
| `#섹션id` | 같은 페이지 내 앵커 이동 |
| `mailto:email@...` | 이메일 클라이언트 실행 |
| `tel:01012345678` | 전화 앱 실행 (모바일) |

### 핵심 원칙

- `<img>`에는 **반드시 `alt` 속성**을 작성합니다. (접근성 + SEO)
- 이미지와 설명을 함께 쓸 때는 `<figure>` + `<figcaption>`으로 묶습니다.
- `<audio>` / `<video>`에 `controls` 속성이 없으면 재생 버튼이 표시되지 않습니다.
- 여러 포맷을 지원하려면 `<source>` 태그를 중첩하여 폴백(fallback)을 제공합니다.

### 실습 파일

| 파일 | 학습 내용 |
|------|-----------|
| `example.html` | img, audio, video, a 종합 예제 |
| `profile_card.html` | 프로필 카드 — 링크 종류별 활용 |

## Chapter 5 — 시맨틱 태그

> 📁 [`semantic_tag/README.md`](./semantic_tag/README.md)

**학습 목표:** 태그 이름만으로 역할을 알 수 있는 시맨틱 태그로 페이지 구조를 설계합니다.

### 시맨틱 vs 논시맨틱

| 구분 | 예시 | 특징 |
|------|------|------|
| **시맨틱** | `<header>`, `<nav>`, `<main>`, `<article>` | 태그 이름이 곧 역할 |
| **논시맨틱** | `<div>`, `<span>` | 태그 이름만으로 역할 파악 불가 |

### 주요 시맨틱 태그

| 태그 | 역할 | 사용 위치 |
|------|------|-----------|
| `<header>` | 페이지/섹션 최상단 — 로고, 제목, 내비게이션 | `<body>` 또는 `<section>` 안 |
| `<nav>` | 내비게이션 — 메뉴 링크 모음 | 주로 `<header>` 안 |
| `<main>` | 문서의 핵심 본문 콘텐츠 | `<body>` 안, **페이지당 1개** |
| `<section>` | 주제별로 연관된 콘텐츠 구역 | `<main>` 안 |
| `<article>` | 독립적으로 배포·재사용 가능한 콘텐츠 | `<section>` 또는 `<main>` 안 |
| `<aside>` | 메인 콘텐츠와 간접 연관된 부가 정보 | `<main>` 옆 (사이드바) |
| `<footer>` | 페이지/섹션 최하단 — 저작권, 연락처 | `<body>` 또는 `<section>` 안 |

### section vs article 선택 기준

| 질문 | 사용할 태그 |
|------|-------------|
| RSS 피드·SNS에 단독으로 공유할 수 있는 콘텐츠인가? | `<article>` |
| 같은 주제의 다른 콘텐츠와 함께 묶이는 구역인가? | `<section>` |
| 단순히 스타일 적용을 위해 묶는 것인가? | `<div>` |

### 핵심 원칙

- 의미 있는 구역에는 `<div>` 대신 시맨틱 태그를 사용합니다. (SEO, 접근성, 가독성 향상)
- `<main>`은 페이지당 단 하나만 사용합니다.
- `<section>`에는 `<h2>`~`<h6>` 제목 태그를 포함하는 것이 권장됩니다.

### 실습 파일

| 파일 | 학습 내용 |
|------|-----------|
| `example.html` | 시맨틱 태그 7종을 활용한 표준 웹 페이지 전체 구조 |

## Chapter 6 — 폼 & 입력 요소

> 📁 [`form_tag/README.md`](./form_tag/README.md)

**학습 목표:** 사용자 입력을 받아 서버로 전달하는 폼(Form) 요소를 올바르게 구성합니다.

### 폼 구성 태그

| 태그 | 역할 |
|------|------|
| `<form>` | 데이터 전송 컨테이너 — `action`(URL), `method`(GET/POST) |
| `<fieldset>` | 관련 입력 요소를 논리적으로 그룹화 |
| `<legend>` | `<fieldset>`의 제목 |
| `<label>` | 입력 필드 이름 — `for` 속성으로 `<input id>`와 연결 |
| `<input>` | 기본 입력 필드 — `type` 속성으로 모양과 기능 변경 |
| `<textarea>` | 여러 줄 텍스트 입력 |
| `<select>` / `<option>` | 드롭다운 선택 메뉴 |
| `<button>` | 제출(`submit`) / 초기화(`reset`) / 커스텀(`button`) |

### input type 정리

| type | 기능 |
|------|------|
| `text` | 일반 한 줄 텍스트 |
| `password` | 마스킹 처리 (●●●) |
| `email` | @ 포함 여부 자동 검증 |
| `tel` | 모바일 숫자 키패드 활성화 |
| `number` | 숫자 전용, min/max/step 지원 |
| `radio` | 단일 선택 (같은 `name` 그룹) |
| `checkbox` | 다중 선택 (같은 `name` 그룹) |
| `file` | 파일 선택 창 |
| `range` | 슬라이더 |
| `date` | 날짜 선택기 |
| `color` | 색상 팔레트 선택기 |

### 핵심 원칙

- `name` 속성이 없으면 서버에서 데이터를 식별할 수 없습니다. (**필수**)
- `<label for="id">`와 `<input id="id">`를 연결하면 라벨 클릭 시 포커스가 이동합니다.
- 비밀번호·개인정보는 반드시 `method="post"`를 사용합니다.
- 필수 항목에는 `required` 속성을 추가해 클라이언트 측 검증을 먼저 처리합니다.

### 실습 파일

| 파일 | 학습 내용 |
|------|-----------|
| `example.html` | 회원가입 양식 — fieldset 3개, input 9종, select, textarea, button |

## 태그 전체 요약표

| 챕터 | 핵심 태그 | 핵심 개념 |
|------|-----------|-----------|
| 1. 텍스트 | `h1~h6`, `p`, `br`, `strong`, `em`, `q`, `blockquote`, `ins`, `del` | 의미 vs 시각적 표현 구분 |
| 2. 블록/인라인 | `div`, `span` | 레이아웃 레벨 차이, 올바른 중첩 |
| 3. 목록/표 | `ul`, `ol`, `dl`, `table`, `thead`, `tbody`, `th`, `td` | 데이터 구조에 맞는 태그 선택 |
| 4. 미디어/링크 | `img`, `audio`, `video`, `source`, `a`, `figure` | alt 속성, 다양한 href 유형 |
| 5. 시맨틱 | `header`, `nav`, `main`, `section`, `article`, `aside`, `footer` | 의미 있는 구조 설계 |
| 6. 폼 | `form`, `input`, `label`, `select`, `textarea`, `button` | name/id 역할 구분, GET vs POST |

---

> **공통 핵심 원칙:** 태그는 **시각적 결과가 아닌 데이터의 의미**를 기준으로 선택하세요.
> 스타일은 CSS로 제어하고, HTML은 구조와 의미에 집중합니다.
> 시맨틱한 HTML은 SEO, 접근성, 유지보수 모두를 향상시킵니다.
