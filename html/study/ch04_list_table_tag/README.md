# HTML 구조와 데이터 표현: 태그 관계, 목록, 표

HTML은 단순한 텍스트가 아니라 태그들의 **계층적 구조**로 이루어져 있습니다.
태그 간의 관계를 이해하고, 목록과 표를 올바르게 사용하는 것이 구조화의 핵심입니다.

## 1. 태그와 요소의 이해

### 핵심 개념

| 개념 | 설명 | 예시 |
|------|------|------|
| **태그(Tag)** | HTML 코드를 구성하는 최소 단위 | `<p>`, `<div>` |
| **요소(Element)** | 여는 태그 + 내용 + 닫는 태그 전체 | `<p>안녕</p>` |
| **속성(Attribute)** | 태그에 추가 정보를 제공 | `<a href="...">` |
| **중첩(Nesting)** | 태그 안에 다른 태그를 포함해 계층 구조 형성 | `<ul><li>항목</li></ul>` |

### 태그 간의 관계 — 가족 관계로 이해하기

HTML은 **트리(Tree) 구조**를 가집니다. 마치 가족 관계도처럼 요소들이 상하 계층으로 연결됩니다.

```
<div>               ← 부모(Parent)
  <h2>제목</h2>     ← 자식(Child) — div 안에 직접 포함됨
  <p>내용</p>       ← 형제(Sibling) — h2와 같은 레벨
</div>
```

| 관계 | 설명 | 비유 |
|------|------|------|
| **부모(Parent)** | 특정 요소를 감싸고 있는 상위 요소 | 나를 품고 있는 상위 폴더 |
| **자식(Child)** | 특정 요소 내부에 포함된 하위 요소 | 폴더 안의 파일 |
| **형제(Sibling)** | 동일한 부모를 가진 같은 레벨의 요소 | 같은 폴더 안의 다른 파일들 |

### 실생활 예시: 목차처럼 생긴 HTML 트리

```
책 (body)
 ├── 1장 (section)
 │     ├── 1장 제목 (h2)
 │     ├── 1절 (article)
 │     │     └── 1절 내용 (p)
 │     └── 2절 (article)
 └── 2장 (section)
       └── 2장 제목 (h2)
```

이렇게 HTML 문서는 눈에 보이지 않는 나무 구조(Tree)로 이루어져 있으며,
이것을 **DOM(Document Object Model)** 이라고 부릅니다.

## 2. 목록 태그 (Lists)

항목을 나열할 때 사용하며, 목적에 따라 세 가지 유형이 있습니다.

### 왜 목록 태그를 써야 할까요?

일반 `<p>` 태그로 항목을 나열할 수도 있지만, 목록 태그를 사용하면:
- 검색 엔진이 "이건 목록이구나"라고 **의미를 파악**할 수 있습니다
- 스크린 리더가 "3개 항목 중 1번째"처럼 **몇 개인지 안내**해줍니다
- CSS로 손쉽게 **스타일을 통일**할 수 있습니다

---

### 순서가 있는 목록 `ol`

단계적 절차나 순위가 중요할 때 사용합니다. 항목 앞에 **번호(1, 2, 3...)** 가 자동으로 매겨집니다.

> **언제 쓰나요?** 순서가 바뀌면 의미가 달라지는 경우입니다.  
> 예) 요리 레시피, 설치 가이드, 순위표

```html
<ol>
  <li>원두를 갈아냅니다.</li>
  <li>뜨거운 물을 붓습니다.</li>
  <li>2분간 기다립니다.</li>
</ol>
```

결과 화면:
```
1. 원두를 갈아냅니다.
2. 뜨거운 물을 붓습니다.
3. 2분간 기다립니다.
```

---

### 순서가 없는 목록 `ul`

순서와 상관없는 항목을 나열합니다. 항목 앞에 **불릿(●)** 이 표시됩니다.

> **언제 쓰나요?** 순서가 달라져도 의미가 같은 경우입니다.  
> 예) 메뉴판, 기능 목록, 네비게이션 메뉴

```html
<ul>
  <li>아메리카노</li>
  <li>카페라떼</li>
  <li>에스프레소</li>
</ul>
```

결과 화면:
```
● 아메리카노
● 카페라떼
● 에스프레소
```

---

### 설명 목록 `dl`

용어(`<dt>`)와 그에 대한 정의/설명(`<dd>`)을 쌍으로 나타냅니다.

> **언제 쓰나요?** 단어의 뜻을 설명하거나, Q&A(FAQ) 형식으로 나열할 때입니다.  
> 예) 용어 사전, FAQ, 상품 스펙 표

```html
<dl>
  <dt>HTML</dt>
  <dd>웹 페이지의 구조를 만드는 언어입니다.</dd>
  <dt>CSS</dt>
  <dd>웹 페이지의 스타일을 담당하는 언어입니다.</dd>
</dl>
```

결과 화면:
```
HTML
    웹 페이지의 구조를 만드는 언어입니다.
CSS
    웹 페이지의 스타일을 담당하는 언어입니다.
```

---

### 중첩 목록 — 목록 안의 목록

`<li>` 안에 `<ul>` 또는 `<ol>`을 넣어 **계층형 목록**을 만들 수 있습니다.
네비게이션 메뉴의 드롭다운이나 카테고리 분류에 자주 사용합니다.

```html
<ul>
  <li>커피
    <ul>              <!-- 중첩 ul: "커피"의 하위 메뉴 -->
      <li>아메리카노</li>
      <li>바닐라라떼</li>
    </ul>
  </li>
  <li>에이드</li>
</ul>
```

결과 화면:
```
● 커피
    ○ 아메리카노
    ○ 바닐라라떼
● 에이드
```

---

### 주의사항

- `<ul>` / `<ol>`의 **직계 자식은 반드시 `<li>`** 여야 합니다.
  ```html
  <!-- ❌ 잘못된 예 -->
  <ul>
    <div>항목1</div>   <!-- ul의 직계 자식이 div — 잘못됨 -->
  </ul>

  <!-- ✅ 올바른 예 -->
  <ul>
    <li>항목1</li>     <!-- ul의 직계 자식은 li -->
  </ul>
  ```
- 순서가 없는 항목에 `<ol>`을, 순서가 있는 항목에 `<ul>`을 쓰지 않습니다.

---

### 목록 태그 비교 요약

| 태그 | 이름 | 마커 | 사용 예시 |
|------|------|:----:|-----------|
| `<ol>` | Ordered List | 1, 2, 3... | 순위, 단계별 절차 |
| `<ul>` | Unordered List | ● | 메뉴, 기능 목록 |
| `<dl>` | Description List | 없음 | 용어 사전, FAQ |
| `<li>` | List Item | — | ol / ul 안의 각 항목 |
| `<dt>` | Description Term | — | dl 안의 용어 |
| `<dd>` | Description Detail | — | dl 안의 설명 |

## 3. 표 태그 (Tables)

복잡한 데이터를 **행(Row)과 열(Column)** 로 구성된 격자 형태로 표현합니다.

### 표는 언제 써야 할까요?

표는 **2차원 데이터**를 표현할 때만 사용합니다.  
즉, "가로 축(열)과 세로 축(행)이 교차하는 지점에 값이 있는 구조"일 때입니다.

| 표를 써야 할 상황 | 표를 쓰면 안 되는 상황 |
|-------------------|------------------------|
| 시간표, 요금표, 명단 | 3단 레이아웃 (헤더/사이드바/본문) |
| 성적표, 스펙 비교표 | 카드 형태의 갤러리 |
| 데이터 통계 | 이미지 나란히 배치 |

---

### 표 구성 태그

| 태그 | 이름 | 역할 |
|------|------|------|
| `<table>` | Table | 표 전체를 감싸는 컨테이너 |
| `<caption>` | Caption | 표의 제목 — `<table>` 바로 아래에 위치 |
| `<tr>` | Table Row | 가로 한 행 |
| `<th>` | Table Header | 제목 셀 — 기본: **굵게 + 가운데 정렬** |
| `<td>` | Table Data | 데이터 셀 — 기본: 왼쪽 정렬 |

### 구조화 태그 — 표를 영역으로 나누기

표가 길어질 경우 의미론적으로 영역을 나눕니다.
스크린 리더가 각 영역을 인식하고, 브라우저가 인쇄 시 머리글을 각 페이지마다 반복 출력합니다.

| 태그 | 역할 | 특징 |
|------|------|------|
| `<thead>` | 머리글 영역 | 컬럼 제목 행을 포함 |
| `<tbody>` | 본문 데이터 영역 | 실제 데이터 행들을 포함 |
| `<tfoot>` | 바닥글 영역 | 합계, 요약 등을 포함 |

### 셀 병합 속성 — 엑셀의 '셀 병합'과 같은 기능

| 속성 | 설명 | 예시 |
|------|------|------|
| `colspan="n"` | n개의 **열(Column)** 을 하나로 병합 | `<td colspan="2">` |
| `rowspan="n"` | n개의 **행(Row)** 을 하나로 병합 | `<td rowspan="3">` |

### 코드 예시

```html
<table>
  <caption>📅 2026년 상반기 코딩 모임 명단</caption>

  <thead>
    <tr>
      <th>성함</th>
      <th>활동 분야</th>
      <th>비고</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>김코딩</td>
      <td>Frontend</td>
      <td>팀장</td>
    </tr>
    <tr>
      <td>이코딩</td>
      <td>Backend</td>
      <td>-</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <!-- colspan="2": "총 인원" 셀이 2개 열을 병합 -->
      <td colspan="2">총 인원</td>
      <td>2명</td>
    </tr>
  </tfoot>
</table>
```

결과 화면 (개념도):
```
┌──────────────────────────────────────────┐
│    📅 2026년 상반기 코딩 모임 명단        │  ← caption
├──────────┬─────────────┬─────────────────┤
│  성함    │  활동 분야  │      비고       │  ← thead (th)
├──────────┼─────────────┼─────────────────┤
│  김코딩  │  Frontend   │     팀장        │  ← tbody (td)
│  이코딩  │  Backend    │      -          │
├──────────┴─────────────┼─────────────────┤
│        총 인원         │      2명        │  ← tfoot (colspan)
└────────────────────────┴─────────────────┘
```

### 주의사항

- `<caption>`은 스크린 리더가 표를 읽기 전에 먼저 읽어줘 접근성에 중요합니다.
  "표가 어떤 내용인지" 미리 알려주는 역할을 합니다.
- `border="1"` 같은 HTML 속성 방식은 구버전(HTML4) 방식입니다. CSS `border`로 처리하세요.

## 4. 표로 레이아웃을 만들면 안 되는 이유

### 과거에는 표로 레이아웃을 만들었습니다

인터넷 초창기에는 CSS가 발달하지 않아, `<table>` 태그로 3단 레이아웃(왼쪽 메뉴 / 본문 / 오른쪽 광고)을 만들곤 했습니다.
오늘날에도 이런 구조를 간혹 볼 수 있지만 **절대로 따라 하면 안 됩니다.**

### 왜 안 될까요?

| 문제 | 설명 |
|------|------|
| **접근성** | 스크린 리더가 표를 "데이터 표"로 인식해 "1행 1열, 1행 2열..." 순서로 읽습니다. 레이아웃을 표로 만들면 시각 장애인에게 완전히 다른 내용으로 전달됩니다. |
| **SEO** | 검색 엔진이 콘텐츠의 의미 구조를 파악하지 못해 검색 순위가 떨어집니다. |
| **유지보수** | 표 안에 표를 중첩하는 구조가 되면 코드가 매우 복잡해져 수정이 어려워집니다. |
| **반응형** | 표는 기본적으로 고정 너비이므로 모바일 환경에 맞게 변형하기 어렵습니다. |

```html
<!-- ❌ 절대 이렇게 하지 마세요 — 표로 만든 레이아웃 -->
<table>
  <tr>
    <td>← 사이드바</td>
    <td>← 본문</td>
    <td>← 광고</td>
  </tr>
</table>

<!-- ✅ 올바른 방식 — 시맨틱 태그 + CSS Flexbox/Grid -->
<main>
  <nav>← 사이드바</nav>
  <section>← 본문</section>
  <aside>← 광고</aside>
</main>
```

## 5. 태그 비교 요약

### 목록 vs 표 선택 기준

| 상황 | 사용할 태그 | 이유 |
|------|-------------|------|
| 순서가 중요한 항목 나열 | `<ol>` | 번호가 의미를 가짐 |
| 순서가 없는 항목 나열 | `<ul>` | 불릿으로 목록임을 표시 |
| 용어와 설명을 쌍으로 표현 | `<dl>` | 키-값 구조에 적합 |
| 행과 열로 구성된 데이터 표현 | `<table>` | 2차원 데이터 전용 |
| 레이아웃 구성 | CSS Flexbox / Grid | 표는 레이아웃 용도 금지 |

### 예제 파일 목록

| 파일 | 학습 내용 |
|------|-----------|
| `list.html` | `ul` / `ol` 및 중첩 목록 구조 |
| `table.html` | `table` 구성 태그 및 `colspan` 셀 병합 |

---

> **핵심 원칙:** 태그는 **시각적 모양이 아닌 데이터의 의미**에 맞게 선택하세요.  
> 순서가 있으면 `<ol>`, 없으면 `<ul>`, 2차원 데이터면 `<table>`입니다.  
> 그리고 **표(table)는 절대 레이아웃 용도로 쓰지 마세요.**
