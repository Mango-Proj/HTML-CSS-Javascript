## 3. 페이지 구성

페이지는 시맨틱 태그로 구획을 나누고, 내비게이션 앵커 링크로 각 섹션을 연결합니다.

```
┌─────────────────────────────────────────────────────────┐
│  <header>  로고(h1) + <nav> 앵커 링크 메뉴               │
│            sticky — 스크롤 시 상단 고정                  │
├─────────────────────────────────────────────────────────┤
│  <section id="hero">  히어로 배너                        │
│    h2 / p / blockquote / a(버튼 링크)                   │
├──────────────────────────────────┬──────────────────────┤
│  <main>                          │  <aside>             │
│  ┌───────────────────────────┐   │  - 공지사항 ul        │
│  │ section#intro             │   │  - 일정 ol           │
│  │  텍스트 강조·인용·편집 태그│   │  - 연락처 dl         │
│  ├───────────────────────────┤   │    a(mailto / tel)   │
│  │ section#curriculum        │   │                      │
│  │  ol(중첩 ul) + dl         │   │  position: sticky    │
│  ├───────────────────────────┤   │  스크롤 시 고정       │
│  │ section#members           │   │                      │
│  │  article + figure + img   │   │                      │
│  ├───────────────────────────┤   │                      │
│  │ section#record            │   │                      │
│  │  table 전체 구조           │   │                      │
│  ├───────────────────────────┤   │                      │
│  │ section#join              │   │                      │
│  │  form 전체 (input 9종)    │   │                      │
│  └───────────────────────────┘   │                      │
├──────────────────────────────────┴──────────────────────┤
│  <footer>  &copy; / a(mailto) / a(tel) / 이용약관        │
└─────────────────────────────────────────────────────────┘
```

## 4. 활용 태그 목록

학습한 6개 챕터의 태그를 모두 사용했습니다.

### ① text_tag

| 태그 | 사용 위치 | 역할 |
|------|-----------|------|
| `<h1>` | `<header>` | 사이트 대제목 (로고) |
| `<h2>` | 각 `<section>` | 섹션 제목 |
| `<h3>` | 멤버 이름, 박스 소제목 | 세부 항목 제목 |
| `<h4>` | `<aside>` 박스 제목 | 최하위 제목 |
| `<p>` | 전체 | 문단 텍스트 |
| `<br>` | hero h2, 소개 문단 | 문단 내 줄 바꿈 |
| `<hr>` | 섹션 사이 | 구분선 |
| `<strong>` | 소개 · footer | 의미적 강조 (중요 내용) |
| `<em>` | 소개 문단 | 의미적 기울임 |
| `<b>` | 소개 문단 | 시각적 굵게 |
| `<i>` | 소개 문단 | 시각적 기울임 |
| `<q>` | 소개 모토 | 짧은 인용구 |
| `<blockquote>` | hero 슬로건 | 긴 인용문 |
| `<del>` | 공지 변경 이력 | 삭제된 내용 (취소선) |
| `<ins>` | 공지 변경 이력 | 추가된 내용 (밑줄) |

### ② block_inline_tag

| 태그 | 사용 위치 | 역할 |
|------|-----------|------|
| `<div>` | 그리드, 래퍼 | 범용 블록 컨테이너 |
| `<span>` | 뱃지, 로고 색상 | 인라인 텍스트 스타일링 |

### ③ list_table_tag

| 태그 | 사용 위치 | 역할 |
|------|-----------|------|
| `<ul>` | 내비, 공지, 세부 항목 | 순서 없는 목록 |
| `<ol>` | 커리큘럼 단계, 일정 | 순서 있는 목록 |
| `<li>` | ul / ol 안 | 각 항목 |
| `<dl>` | 개념 사전, 연락처 | 용어-설명 목록 |
| `<dt>` | dl 안 | 용어 |
| `<dd>` | dl 안 | 설명 |
| `<table>` | 학습 기록 섹션 | 표 컨테이너 |
| `<caption>` | table 안 | 표 제목 |
| `<thead>` | table 안 | 머리글 영역 |
| `<tbody>` | table 안 | 본문 데이터 영역 |
| `<tfoot>` | table 안 | 바닥글 (합계) 영역 |
| `<tr>` | table 안 | 행 |
| `<th>` | thead 안 | 제목 셀 |
| `<td>` | tbody / tfoot 안 | 데이터 셀 |
| `colspan` | tfoot td | 열 병합 속성 |

### ④ media_tag

| 태그 / 속성 | 사용 위치 | 역할 |
|-------------|-----------|------|
| `<img src alt width height>` | 멤버 프로필 | 이미지 삽입 |
| `<figure>` | 멤버 카드 | 이미지 + 설명 묶음 |
| `<figcaption>` | 멤버 카드 | 이미지 설명 |
| `<a href target="_blank">` | 멤버 SNS 링크 | 외부 링크 (새 탭) |
| `<a href="#섹션id">` | 내비, 히어로 버튼 | 앵커 링크 (페이지 내 이동) |
| `<a href="mailto:...">` | aside, footer | 메일 클라이언트 실행 |
| `<a href="tel:...">` | aside, footer | 전화 앱 실행 |

### ⑤ semantic_tag

| 태그 | 역할 |
|------|------|
| `<header>` | 페이지 상단 — 로고, 내비게이션 |
| `<nav>` | 앵커 링크 메뉴 |
| `<main>` | 핵심 본문 영역 (페이지당 1개) |

| `<footer>` | 페이지 하단 — 저작권, 링크 |

## 5. 섹션별 학습 포인트

### header + nav — 앵커 링크

```html
<nav>
  <ul>
    <li><a href="#intro">소개</a></li>     <!-- #intro → section id="intro" -->
    <li><a href="#join">가입 신청</a></li>
  </ul>
</nav>
```

- `href="#섹션id"` : 같은 페이지 내 특정 위치로 스크롤 이동 (앵커 링크)
- `position: sticky` + `scroll-behavior: smooth` : 부드러운 고정 헤더

---

### section#intro — 텍스트 태그 비교

```html
<strong>프론트엔드 개발자</strong>  → 의미적으로 중요 (스크린 리더 강조)
<em>매주 목요일 저녁 8시</em>       → 의미적 기울임 강조
<b>발표 + 코드 리뷰</b>             → 시각적 굵게 (의미 없음)
<i>"배운 것을 가르치는 것"</i>      → 시각적 기울임 (의미 없음)
<q>완벽한 코드보다 동작하는 코드</q>→ 짧은 인용구 (따옴표 자동 삽입)
<del>매주 화요일 7시</del>          → 삭제된 내용 (취소선)
<ins>매주 목요일 8시</ins>          → 추가된 내용 (밑줄)
```

---

### section#curriculum — 목록 중첩 구조

```html
<ol>                    ← 순서 있는 커리큘럼
  <li>HTML 기초
    <ul>                ← 중첩 ul: 세부 항목
      <li>텍스트 구조화 태그</li>
      <li>블록 / 인라인 요소</li>
    </ul>
  </li>
  <li>CSS 기초</li>
</ol>
```

- `<ol>` : 단계(순서)가 중요한 커리큘럼
- `<ul>` : 순서가 없는 세부 항목 (ol 안에 중첩)
- `<dl>` : 용어-설명 쌍으로 개념 사전 표현

---

### section#record — 표 전체 구조

```html
<table>
  <caption>표 제목 — 스크린 리더가 먼저 읽음</caption>
  <thead>               ← 머리글 (컬럼 이름)
    <tr><th>회차</th><th>날짜</th>...</tr>
  </thead>
  <tbody>               ← 본문 데이터
    <tr><td>1회</td><td>2026.01.09</td>...</tr>
  </tbody>
  <tfoot>               ← 바닥글 (합계)
    <tr>
      <td colspan="4">총 진행 / 전체 계획</td>  ← 열 4개 병합
      <td>6회 / 12회</td>
    </tr>
  </tfoot>
</table>
```

---

### section#join — 폼 핵심 속성 비교

| 속성 | 역할 | 예시 |
|------|------|------|
| `name` | 서버 전송 시 키(Key) — 필수 | `name="email"` |
| `id` | label의 `for`와 연결 | `id="email"` |
| `required` | 비워두면 제출 차단 | `required` |
| `placeholder` | 힌트 텍스트 | `placeholder="입력하세요"` |
| `value` | 초기값 / 전송값 | `value="#5b8dee"` |
| `checked` | radio/checkbox 기본 선택 | `checked` |
| `selected` | option 기본 선택 | `selected` |
| `min` / `max` | number 입력 범위 | `min="0" max="20"` |

---

### aside — 사이드바 구성

```html
<aside>
  <!-- ul: 순서 없는 공지 목록 -->
  <ul> <li>공지 내용</li> </ul>

  <!-- dl: 연락처 항목명-내용 쌍 -->
  <dl>
    <dt>이메일</dt>
    <dd><a href="mailto:...">주소</a></dd>   <!-- 메일 앱 실행 -->
    <dt>전화</dt>
    <dd><a href="tel:...">번호</a></dd>      <!-- 전화 앱 실행 -->
  </dl>
</aside>
```

## 6. 미리보기

브라우저에서 `index.html`을 열면 아래 순서로 내용이 표시됩니다.

```
[헤더 내비게이션] ← 스크롤 시 상단 고정 (sticky)
[히어로 배너]     ← 슬로건 + 가입 신청 버튼
[소개]            ← 텍스트 강조·인용·편집 태그 비교
[커리큘럼]        ← ol 중첩 목록 + dl 개념 사전
[멤버]            ← article + figure + img 프로필 카드
[학습 기록]       ← table (thead/tbody/tfoot/colspan)
[가입 신청]       ← form (input 9종 + select + textarea)
[사이드바]        ← aside (공지 ul / 일정 ol / 연락처 dl)
[푸터]            ← &copy; + a(mailto) + a(tel)
```
