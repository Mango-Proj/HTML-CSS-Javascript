# HTML 텍스트 구조화 가이드

> 태그는 **시각적 디자인이 아닌 의미와 구조**를 기준으로 선택하세요.
> 디자인은 언제나 CSS가 담당합니다.

웹 페이지의 가독성과 접근성을 높이려면 제목과 본문을 명확히 구분하는 구조화가 필수입니다.
올바른 구조화는 사용자뿐 아니라 검색 엔진(SEO)과 스크린 리더(Screen Reader)가 페이지를 정확히 해석하는 데도 큰 영향을 미칩니다.

## 1. 제목 태그 `h1`~`h6`

주제나 섹션의 타이틀을 나타낼 때 사용합니다. `<h1>`부터 `<h6>`까지 총 **6단계**의 계층을 제공합니다.

### 태그별 특징

| 태그 | 기본 크기 | 역할 |
|------|:---------:|------|
| `<h1>` | 가장 큼 | 문서 전체의 대제목 — 페이지당 **1개** 권장 |
| `<h2>` | 큼 | 주요 섹션의 제목 |
| `<h3>` | 중간 | 세부 섹션의 제목 |
| `<h4>` | 약간 작음 | 소제목 |
| `<h5>` | 작음 | 더 작은 소제목 |
| `<h6>` | 가장 작음 | 가장 낮은 단계의 제목 |

### 코드 예시

```html
<h1>h1: 가장 중요한 대제목</h1>
<h2>h2: 섹션의 주제</h2>
<h3>h3: 세부 섹션의 제목</h3>
<h4>h4: 소제목</h4>
<h5>h5: 더 작은 소제목</h5>
<h6>h6: 가장 낮은 단계의 제목</h6>
```

### 주의사항

- **`<h1>`은 페이지당 한 번만** — 문서의 핵심 주제이므로 단 한 번만 사용합니다.
- **계층 순서 준수** — 단계를 건너뛰지 않고 순서대로 사용합니다.

  ```
  ✅ 올바른 예: h1 → h2 → h3
  ❌ 잘못된 예: h1 → h3  (h2 건너뜀)
  ```

- **디자인 목적 사용 금지** — 글씨 크기 조절이 목적이라면 CSS를 사용하세요.
- **SEO 활용** — `<h1>`에 페이지의 핵심 키워드를 담으면 검색 순위에 긍정적인 영향을 줍니다.

## 2. 문단 태그 `p`

본문의 문단(단락)을 구분할 때 사용하며, **Paragraph**의 약자입니다.

### 특징

- 브라우저가 `<p>` 앞뒤에 자동으로 **위·아래 여백(Margin)** 을 생성해 문단을 분리합니다.
- **블록 레벨 요소**로, 항상 새 줄에서 시작하고 가로 전체 너비를 차지합니다.
- 하나의 주제·맥락으로 묶인 내용만 하나의 `<p>`로 감쌉니다.

### 코드 예시

```html
<p>동해 물과 백두산이 마르고 닳도록 하느님이 보우하사 우리나라 만세...</p>
<p>남산 위에 저 소나무 철갑을 두른 듯 바람 서리 불변함은 우리 기상일세...</p>
```

### `<p>`를 사용해야 하는 이유

| 관점 | 효과 |
|------|------|
| SEO | 검색 엔진이 본문 텍스트를 정확히 인식 |
| 접근성 | 스크린 리더가 단락 구분을 사용자에게 전달 |
| 유지보수 | CSS로 문단 스타일을 일괄 관리 가능 |

## 3. 줄 바꿈 태그 `br`

문단 **내에서** 강제로 줄을 바꿀 때 사용하며, **Line Break**의 약자입니다.

### 특징

- 종료 태그가 없는 **빈 태그(Empty tag / Void element)** 입니다.
- 새로운 블록을 생성하지 않고 현재 줄만 끊습니다.
- 시(詩) 구절, 우편 주소처럼 **단락은 같지만 줄이 나뉘어야** 하는 경우에 적합합니다.

### 코드 예시

```html
<!-- 주소: 한 문단이지만 줄을 나눠야 하는 경우 -->
<p>
  서울특별시 중구 세종대로 110<br>
  (태평로1가 31)
</p>

<!-- 시 구절: 행(行) 단위 줄 바꿈 -->
<p>
  나 보기가 역겨워<br>
  가실 때에는<br>
  말없이 고이 보내 드리오리다.
</p>
```

### 주의사항

문단 사이 간격을 벌리기 위해 `<br>`을 연속으로 사용하면 안 됩니다.
문단 구분은 `<p>` 태그로, 간격 조절은 CSS `margin`으로 처리합니다.

```html
<!-- ❌ 잘못된 예: <br>로 간격 조절 -->
첫 번째 문단입니다.<br><br><br>
두 번째 문단입니다.

<!-- ✅ 올바른 예: <p>로 문단 구분 -->
<p>첫 번째 문단입니다.</p>
<p>두 번째 문단입니다.</p>
```

## 4. 태그 비교 요약

| 태그 | 의미 | 용도 | 레벨 | 빈 태그 |
|------|------|------|:----:|:-------:|
| `<h1>`~`<h6>` | Heading | 계층적 제목 | 블록 | ✗ |
| `<p>` | Paragraph | 문단 구분 | 블록 | ✗ |
| `<br>` | Line Break | 문단 내 줄 바꿈 | 인라인 | ✓ |

## 5. 실전 구조화 예제

제목, 문단, 줄 바꿈 태그를 함께 사용한 실제 문서 구조 예시입니다.

```html
<section>
  <h1>나의 학습 일지</h1>            <!-- 문서 대제목 (1개만) -->

  <h2>오늘 배운 내용: HTML 기초</h2>  <!-- 주요 섹션 -->
  <p>
    오늘은 HTML의 텍스트 구조화 태그들을 배웠습니다.<br>
    제목 태그와 문단 태그의 차이점을 확실히 알게 되었습니다.
  </p>

  <h3>주의할 점</h3>                 <!-- 세부 항목 -->
  <p>
    디자인을 위해 h 태그를 마음대로 골라 쓰면 안 됩니다.<br>
    구조에 맞는 태그 선택이 무엇보다 중요합니다.
  </p>
</section>
```

### 계층 구조 트리

```
section
└── h1  ← 문서 대제목
    └── h2  ← 주요 섹션
        ├── p   ← 본문 문단 (내부에 br 사용 가능)
        └── h3  ← 세부 항목
            └── p   ← 본문 문단
```
