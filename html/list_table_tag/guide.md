# HTML 구조와 데이터 표현: 태그 관계, 목록, 표

HTML은 단순한 텍스트가 아니라 태그들의 **계층적 구조**로 이루어져 있습니다.
태그 간의 관계를 이해하고, 목록과 표를 올바르게 사용하는 것이 구조화의 핵심입니다.

## 1. 태그와 요소의 이해

### 핵심 개념

| 개념 | 설명 | 예시 |
|------|------|------|
| **태그(Tag)** | HTML 코드를 구성하는 최소 단위 | `<p>`, `<div>` |
| **요소(Element)** | 여는 태그 + 내용 + 닫는 태그 전체 | `<p>안녕</p>` |
| **속성(Attribute)** | 태그에 추가 정보를 제공 | `<a href="...">` |
| **중첩(Nesting)** | 태그 안에 다른 태그를 포함해 계층 구조 형성 | `<ul><li>항목</li></ul>` |

### 태그 간의 관계

HTML은 **트리(Tree) 구조**를 가집니다.

```
<div>               ← 부모(Parent)
  <h2>제목</h2>     ← 자식(Child) / div의 관점에서
  <p>내용</p>       ← 형제(Sibling) / h2와 같은 레벨
</div>
```

| 관계 | 설명 |
|------|------|
| **부모(Parent)** | 특정 요소를 감싸고 있는 상위 요소 |
| **자식(Child)** | 특정 요소 내부에 포함된 하위 요소 |
| **형제(Sibling)** | 동일한 부모를 가진 같은 레벨의 요소 |

## 2. 목록 태그 (Lists)

항목을 나열할 때 사용하며, 목적에 따라 세 가지 유형이 있습니다.

### 순서가 있는 목록 `ol`

단계적 절차나 순위가 중요할 때 사용합니다. 항목 앞에 **번호(1, 2, 3...)** 가 자동으로 매겨집니다.

```html
<ol>
  <li>원두를 갈아냅니다.</li>
  <li>뜨거운 물을 붓습니다.</li>
</ol>
```

### 순서가 없는 목록 `ul`

순서와 상관없는 항목을 나열합니다. 항목 앞에 **불릿(●)** 이 표시됩니다.

```html
<ul>
  <li>아메리카노</li>
  <li>카페라떼</li>
</ul>
```

### 설명 목록 `dl`

용어(`<dt>`)와 그에 대한 정의/설명(`<dd>`)을 쌍으로 나타냅니다.

```html
<dl>
  <dt>HTML</dt>
  <dd>웹 페이지의 구조를 만드는 언어입니다.</dd>
  <dt>CSS</dt>
  <dd>웹 페이지의 스타일을 담당하는 언어입니다.</dd>
</dl>
```

### 중첩 목록

`<li>` 안에 `<ul>` 또는 `<ol>`을 넣어 **계층형 목록**을 만들 수 있습니다.

```html
<ul>
  <li>커피
    <ul>              <!-- 중첩 ul: "커피"의 하위 메뉴 -->
      <li>아메리카노</li>
      <li>바닐라라떼</li>
    </ul>
  </li>
  <li>에이드</li>
</ul>
```

### 주의사항

- `<ul>` / `<ol>`의 **직계 자식은 반드시 `<li>`** 여야 합니다.
- 순서가 없는 항목에 `<ol>`을 쓰거나, 순서가 있는 항목에 `<ul>`을 쓰지 마세요.

### 목록 태그 비교

| 태그 | 이름 | 마커 | 사용 예시 |
|------|------|:----:|-----------|
| `<ol>` | Ordered List | 1, 2, 3... | 순위, 단계별 절차 |
| `<ul>` | Unordered List | ● | 메뉴, 기능 목록 |
| `<dl>` | Description List | 없음 | 용어 사전, FAQ |
| `<li>` | List Item | — | ol / ul 안의 각 항목 |
| `<dt>` | Description Term | — | dl 안의 용어 |
| `<dd>` | Description Detail | — | dl 안의 설명 |

## 3. 표 태그 (Tables)

복잡한 데이터를 **행(Row)과 열(Column)** 로 구성된 격자 형태로 표현합니다.

### 표 구성 태그

| 태그 | 이름 | 역할 |
|------|------|------|
| `<table>` | Table | 표 전체를 감싸는 컨테이너 |
| `<caption>` | Caption | 표의 제목 — `<table>` 바로 아래에 위치 |
| `<tr>` | Table Row | 가로 한 행 |
| `<th>` | Table Header | 제목 셀 — 기본: **굵게 + 가운데 정렬** |
| `<td>` | Table Data | 데이터 셀 — 기본: 왼쪽 정렬 |

### 구조화 태그

표가 길어질 경우 의미론적으로 영역을 나눕니다.

| 태그 | 역할 | 특징 |
|------|------|------|
| `<thead>` | 머리글 영역 | 컬럼 제목 행을 포함 |
| `<tbody>` | 본문 데이터 영역 | 실제 데이터 행들을 포함 |
| `<tfoot>` | 바닥글 영역 | 합계, 요약 등을 포함 |

### 셀 병합 속성

| 속성 | 설명 | 예시 |
|------|------|------|
| `colspan="n"` | n개의 **열(Column)** 을 하나로 병합 | `<td colspan="2">` |
| `rowspan="n"` | n개의 **행(Row)** 을 하나로 병합 | `<td rowspan="3">` |

### 코드 예시

```html
<table>
  <caption>📅 2026년 상반기 코딩 모임 명단</caption>

  <thead>
    <tr>
      <th>성함</th>
      <th>활동 분야</th>
      <th>비고</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>김코딩</td>
      <td>Frontend</td>
      <td>팀장</td>
    </tr>
    <tr>
      <td>이코딩</td>
      <td>Backend</td>
      <td>-</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <!-- colspan="2": "총 인원" 셀이 2개 열을 병합 -->
      <td colspan="2">총 인원</td>
      <td>2명</td>
    </tr>
  </tfoot>
</table>
```

### 주의사항

- **표는 데이터 표현 전용** — 레이아웃 구성 목적으로 `<table>`을 사용하면 안 됩니다.
- `<caption>`은 스크린 리더가 표를 읽기 전에 먼저 읽어줘 접근성에 중요합니다.
- `border="1"` 같은 HTML 속성 방식은 구버전(HTML4) 방식입니다. CSS `border`로 처리하세요.

## 4. 태그 비교 요약

### 목록 vs 표 선택 기준

| 상황 | 사용할 태그 |
|------|-------------|
| 순서가 중요한 항목 나열 | `<ol>` |
| 순서가 없는 항목 나열 | `<ul>` |
| 용어와 설명을 쌍으로 표현 | `<dl>` |
| 행과 열로 구성된 데이터 표현 | `<table>` |

### 예제 파일 목록

| 파일 | 학습 내용 |
|------|-----------|
| `list.html` | `ul` / `ol` 및 중첩 목록 구조 |
| `table.html` | `table` 구성 태그 및 `colspan` 셀 병합 |

---

> **핵심 원칙:** 태그는 **시각적 모양이 아닌 데이터의 의미**에 맞게 선택하세요.
> 순서가 있으면 `<ol>`, 없으면 `<ul>`, 2차원 데이터면 `<table>`입니다.
