          05 Fetch 에러 처리 패턴

          fetch()는 HTTP 에러(404, 500)에서도 Promise를 reject하지 않습니다.
          반드시 response.ok 또는 response.status를 직접 확인해야 합니다.

          ✅ 정상 요청

          fetch('…/todos/1')

          ⚠️ HTTP 404 에러

          fetch('…/todos/99999')

          🔴 네트워크 에러

          fetch('https://invalid.domain.xyz')

          response.ok 체크 패턴

          const res = await fetch(url);

// ❌ fetch는 404여도 reject하지 않음
// → response.ok를 반드시 직접 확인해야 함

if (!res.ok) {
  throw new Error(`HTTP 오류: ${res.status}`);
}

const data = await res.json();
