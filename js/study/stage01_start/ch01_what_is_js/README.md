## 1. 자바스크립트란?

자바스크립트(JavaScript)는 **동적 웹 페이지**를 만드는 데 사용하는 프로그래밍 언어입니다.

> **동적 요소**: 사용자의 동작(클릭, 입력 등)에 따라 화면이 변화하는 것을 의미합니다.

여러분이 매일 쓰는 웹 서비스에는 이미 자바스크립트가 가득합니다.

- **쇼핑몰 장바구니**: 상품을 클릭하면 숫자가 올라가는 것 → JS
- **SNS 좋아요 버튼**: 클릭하면 색이 바뀌고 숫자가 변하는 것 → JS
- **검색 자동완성**: 글자를 치면 아래 추천 목록이 실시간으로 나타나는 것 → JS
- **카카오맵 드래그**: 지도를 마우스로 끌어다 이동하는 것 → JS

HTML 만으로는 이런 기능을 만들 수 없습니다. **사용자가 무언가를 했을 때 화면을 바꿔주는 역할**이 바로 자바스크립트입니다.

### 웹 개발의 3대 핵심 언어

| 언어 | 역할 | 비유 |
|------|------|------|
| HTML | 구조 (뼈대) | 건물의 기둥과 벽 |
| CSS | 디자인 (외형) | 도색과 인테리어 |
| JavaScript | 동작 (기능) | 전기·수도·엘리베이터 |

아무리 예쁜 건물(CSS)도, 튼튼한 뼈대(HTML)도, 전기와 수도(JS)가 없으면 사람이 살 수 없습니다. 세 가지가 함께 있어야 완성된 웹 페이지가 됩니다.

```
사용자 ──클릭/입력──▶ JavaScript ──변경──▶ HTML/CSS ──업데이트──▶ 화면
```

## 2. 자바스크립트의 역사

| 연도 | 사건 |
|------|------|
| 1995 | Brendan Eich가 넷스케이프에서 10일 만에 개발 |
| 1995 | 이름 변천: `Mocha` → `LiveScript` → `JavaScript` |
| 1997 | ECMAScript(ECMA-262) 표준 1판 발표 |
| 2009 | ES5 — `forEach`, `map`, `JSON` 지원 추가 |
| 2009 | **Node.js** 등장 — 서버에서도 JS 실행 가능 |
| 2015 | **ES6(ES2015)** — `let/const`, 화살표 함수, 클래스 등 대규모 업데이트 |
| 매년 | ES2016 ~ ES2024 — 매년 새 표준 발표 |

> **Java 와 JavaScript 는 전혀 다른 언어입니다.**  
> 당시 인기 있던 Java 의 이름을 마케팅 목적으로 빌린 것일 뿐입니다.  
> "햄버거"와 "햄스터"가 이름이 비슷하지만 전혀 다른 것처럼, Java 와 JavaScript 도 사실상 관계가 없습니다.

## 3. 자바스크립트의 실행 방식

자바스크립트 코드는 브라우저(또는 Node.js)가 읽고 바로 실행합니다. 이 역할을 하는 것이 **JS 엔진**입니다.

```
소스 코드(.js)
    ↓
인터프리터(Interpreter)
  - 한 줄씩 읽고 즉시 실행
  - 에러 발생 시 즉각 파악
    ↓
JIT 컴파일러(Just-In-Time Compiler)
  - 자주 실행되는 코드를 기계어로 컴파일
  - 반복 실행 구간의 성능을 향상
    ↓
실행 결과
```

- **인터프리터**: 통역사처럼 코드를 한 줄씩 읽어 즉시 실행합니다. 빠르게 시작할 수 있지만, 같은 코드를 반복할 때마다 다시 읽습니다.
- **JIT 컴파일러**: "이 코드 자주 쓰이네?" 싶은 부분을 미리 기계어로 번역해 두어 속도를 높입니다.

### 주요 JS 엔진

| 엔진 | 사용처 |
|------|--------|
| **V8** | Google Chrome, Node.js |
| **SpiderMonkey** | Mozilla Firefox |
| **JavaScriptCore** | Safari |
| **Chakra** | 구 Microsoft Edge |

현대 JS 엔진(V8 등)은 인터프리터 + JIT 컴파일러를 함께 사용해  
**빠른 응답성과 고성능을 동시에** 제공합니다.

## 5. 브라우저 내장 함수

브라우저에는 JS 엔진이 내장되어 있어 개발자 도구(F12 → Console)에서 바로 실습할 수 있습니다.  
처음 코딩을 배울 때 가장 많이 쓰는 기능들입니다.

### console 메서드

`console`은 코드가 제대로 동작하는지 확인하기 위해 메시지를 출력하는 도구입니다.  
마치 요리 중간중간 간을 보는 것처럼, 코드 중간에 끼워 넣어 값을 확인하는 데 사용합니다.

```js
console.log('일반 메시지');          // 흰색/검은색 — 가장 많이 쓰임
console.warn('경고 메시지');         // 노란색 + 경고 아이콘
console.error('에러 메시지');        // 빨간색 + 에러 아이콘
console.info('정보 메시지');         // 파란색 아이콘
console.table([{ name: 'Alice' }]);  // 표 형태로 출력 — 여러 데이터 비교에 유용
```

### 대화창 함수

사용자와 간단하게 소통할 때 쓰는 브라우저 내장 팝업 함수들입니다.  
버튼을 눌렀을 때 "정말 삭제하시겠습니까?" 같은 확인창을 본 적 있다면, 그것이 바로 `confirm`입니다.

| 함수 | 설명 | 반환값 |
|------|------|--------|
| `alert(msg)` | 메시지 경고창 (확인 버튼만 있음) | `undefined` |
| `confirm(msg)` | 확인/취소 선택창 | `true` / `false` |
| `prompt(msg, 기본값)` | 텍스트 입력창 | 입력 문자열 / `null` |

```js
// alert — 사용자에게 알림 표시 (확인 버튼 클릭 전까지 대기)
alert('안녕하세요!');

// confirm — 사용자의 선택을 받음 (확인 → true, 취소 → false)
const answer = confirm('계속 하시겠습니까?');
if (answer) {
    console.log('확인 선택 → 계속 진행');
} else {
    console.log('취소 선택 → 중단');
}

// prompt — 사용자가 텍스트를 직접 입력 (취소 누르면 null 반환)
const name = prompt('이름을 입력하세요', '홍길동');
if (name !== null) {
    console.log(`안녕하세요, ${name}님!`);
}
```

> **주의**: `alert`, `confirm`, `prompt` 는 실행 중 스크립트를 차단(블로킹)합니다.  
> 팝업이 떠 있는 동안 페이지의 모든 동작이 멈추기 때문에, 실무에서는  
> 모달(modal) UI 컴포넌트를 대신 사용하는 것이 일반적입니다.

## 6. DOM — 자바스크립트가 HTML을 다루는 방법

**DOM(Document Object Model)**: 브라우저가 HTML 문서를 읽어 만든 트리 구조의 객체 모델입니다.

브라우저는 HTML 파일을 읽으면 "아, `<h1>` 태그가 있고, 그 아래 `<p>` 태그가 있구나" 하면서 계층 구조를 머릿속(메모리)에 그림으로 그려 저장합니다. 이 그림이 바로 DOM입니다.  
자바스크립트는 이 DOM을 통해 HTML 요소를 찾고, 내용·스타일·속성을 동적으로 바꿉니다.

```
HTML 문서
└── document
    └── html
        ├── head
        └── body
            ├── h1
            ├── p
            └── button
```

```js
// 요소 선택 — HTML에서 원하는 부분을 "집어오기"
const el   = document.getElementById('my-id');      // id 로 하나 선택
const el2  = document.querySelector('.my-class');   // CSS 선택자로 첫 번째
const list = document.querySelectorAll('li');       // CSS 선택자로 전체 (배열처럼 반환)

// 내용 변경 — 텍스트나 HTML을 교체
el.textContent = '새 텍스트';    // 텍스트만 변경 (안전)
el.innerHTML   = '<b>굵게</b>'; // HTML 포함 변경 (XSS 주의)

// 스타일 변경 — CSS 속성을 JS로 조작
el.style.color      = 'red';
el.style.fontSize   = '24px';

// 이벤트 등록 — 사용자 동작에 반응하게 만들기
// '클릭했을 때 이 함수를 실행해라'는 명령
el.addEventListener('click', function () {
    console.log('클릭 이벤트 발생!');
});
```

## 7. HTML에 자바스크립트 연결하기

JS 코드를 브라우저가 실행하려면 HTML 파일과 연결해야 합니다. 세 가지 방법이 있습니다.

### 방법 1. 인라인 (비권장)

```html
<button onclick="console.log('클릭!')">버튼</button>
```

HTML 태그 안에 JS를 직접 쓰는 방식입니다.  
빠르게 테스트할 때는 편하지만, 코드가 길어지면 HTML과 JS가 뒤섞여 어느 것도 읽기 어려워집니다. 한 문서에 한식·중식·양식 레시피를 뒤섞어 적어 놓은 것과 같습니다.

### 방법 2. 내부 스크립트

```html
<body>
    <span>내용</span>

    <!-- body 하단에 script 태그 작성 -->
    <script>
        console.log('내부 스크립트 실행');
    </script>
</body>
```

HTML 파일 내부에 `<script>` 태그로 JS를 작성합니다. 파일 하나로 관리는 편하지만, 파일이 길어지면 가독성이 떨어집니다.

### 방법 3. 외부 파일 연결 (권장) ✅

```html
<body>
    <span>내용</span>

    <!-- HTML 요소가 모두 로드된 후 실행되도록 body 맨 아래에 위치 -->
    <script src="script.js"></script>
</body>
```

JS를 별도 파일(`.js`)에 작성하고 연결하는 방식입니다.  
HTML은 HTML대로, JS는 JS대로 역할이 분리되어 유지보수하기 훨씬 편합니다.  
여러 HTML 파일에서 같은 JS 파일을 공유할 수도 있고, 브라우저가 파일을 캐싱해 속도도 빠릅니다.

| 방식 | 장점 | 단점 |
|------|------|------|
| 인라인 | 빠른 테스트 | HTML/JS 혼재, 유지보수 어려움 |
| 내부 스크립트 | 별도 파일 불필요 | 파일이 길어지면 가독성 저하 |
| **외부 파일** | 재사용, 캐싱, 분리 명확 | 별도 파일 관리 필요 |

### `<script>` 위치의 중요성

HTML은 위에서 아래로 순서대로 읽힙니다. `<script>` 태그를 만나면 JS를 실행하고 나서야 그 아래 HTML을 읽습니다.  
만약 `<head>` 안에 `<script>`를 배치하면, 아직 버튼이나 텍스트가 그려지지도 않은 상태에서 JS가 실행됩니다.  
이 경우 JS가 "버튼을 찾아라"는 명령을 실행해도 버튼이 아직 없어서 `null`이 반환됩니다.

```
브라우저 HTML 파싱 순서:
<head> 읽기 → ... → <body> 내용 → <script> → 완료

⚠️ head 에 script 배치 시:
  HTML 요소가 아직 없는 상태에서 JS 가 실행 →
  getElementById 등이 null 반환 → 오류 발생

✅ body 하단에 script 배치 시:
  모든 HTML 요소가 생성된 후 실행 → 안전
```

## 8. async / defer — 스크립트 로딩 최적화

외부 스크립트 파일은 다운로드하는 데 시간이 걸립니다. 기본적으로 `<script>` 를 만나면 HTML 파싱을 멈추고 JS를 다운로드·실행한 뒤 다시 HTML을 읽습니다. 이 "멈춤" 현상을 줄이기 위해 `defer` 와 `async` 속성이 생겼습니다.

```html
<!-- defer: HTML 파싱과 병렬로 다운로드, 파싱 완료 후 순서대로 실행 -->
<script src="app.js" defer></script>

<!-- async: HTML 파싱과 병렬로 다운로드, 완료 즉시 실행 (순서 보장 안 됨) -->
<script src="analytics.js" async></script>
```

| 속성 | 다운로드 | 실행 시점 | 실행 순서 | 사용 권장 |
|------|---------|---------|---------|---------|
| 없음 | 순차 | 즉시 (HTML 차단) | 순서 보장 | body 하단 배치 |
| `defer` | 병렬 | HTML 파싱 완료 후 | 순서 보장 ✅ | 일반 스크립트 |
| `async` | 병렬 | 다운로드 완료 즉시 | 보장 안 됨 | 독립적 스크립트 (광고, 통계) |

- **defer**: "다운로드는 미리 받아두되, HTML 다 읽고 나서 실행해줘." → 순서가 보장되므로 일반 스크립트에 권장
- **async**: "다운로드 끝나면 그냥 바로 실행해." → 순서가 보장되지 않으므로 다른 코드와 연관 없는 독립적인 스크립트(구글 애널리틱스, 광고 스크립트 등)에 적합

```
일반 script:  HTML ─██(차단)─── 실행 ──▶ HTML 재개
defer:        HTML ──────────────────────▶ 파싱 완료 후 실행
async:        HTML ────────────────────▶
              다운로드 완료 ──즉시 실행──▶
```

## 핵심 요약

```
JS 역할:    HTML(구조) + CSS(디자인) + JS(동작) 중 동작 담당
실행 방식:  인터프리터 + JIT 컴파일러 혼합 (V8 엔진)
연결 방법:  외부 파일(<script src="...">)을 body 하단에 배치 권장
DOM:        JS 가 HTML 요소를 읽고 변경할 수 있게 해주는 객체 모델
            → HTML을 "찾고" → "내용/스타일을 바꾸고" → "이벤트를 등록"

내장 함수:
  console.log()  → 개발자 도구 콘솔에 출력 (디버깅 핵심)
  alert()        → 경고 팝업 (확인만)
  confirm()      → 확인/취소 팝업 → boolean 반환
  prompt()       → 텍스트 입력 팝업 → 문자열 반환

스크립트 로딩:
  기본           → HTML 차단 (body 하단에 배치)
  defer          → 병렬 다운로드, HTML 완료 후 실행 (순서 보장, 권장)
  async          → 병렬 다운로드, 즉시 실행 (독립적 스크립트에만)
```
