# Ch02. 데이터 타입과 논리 연산

## 학습 목표

- 원시 타입과 참조 타입의 차이를 메모리 관점에서 설명할 수 있다
- 얕은 복사와 깊은 복사의 차이를 이해하고 활용할 수 있다
- `||` 와 `??` 의 차이를 구분하고 적절히 사용할 수 있다
- 옵셔널 체이닝 `?.` 으로 안전하게 속성에 접근할 수 있다
- 형 변환과 `==` vs `===` 의 차이를 이해한다

## 1. 원시 타입 vs 참조 타입 (`ref_type.js`)

데이터 타입은 크게 두 가지로 나뉩니다.  
이 차이를 모르면 "분명히 값을 바꿨는데 왜 원본도 바뀌지?" 하는 버그를 자주 만납니다.

### 종류

| 구분 | 타입 종류 | 저장 방식 |
|------|----------|---------|
| **원시 타입** | `number`, `string`, `boolean`, `null`, `undefined`, `symbol`, `bigint` | 값 자체를 직접 저장 |
| **참조 타입** | `object`, `array`, `function` | 값의 메모리 주소(참조)를 저장 |

### 원시 타입 — 값 복사

원시 타입은 복사할 때 **값 자체**가 복사됩니다. 서로 완전히 독립적입니다.

> **비유:** 파일을 복사해서 붙여넣기 하는 것과 같습니다.  
> 복사본을 수정해도 원본은 변하지 않습니다.

```js
let a = 10;
let b = a;   // 값이 복사됨 → 독립적인 10

b = 20;
console.log(a); // 10 — a는 변하지 않음
console.log(b); // 20
```

```
메모리:
  a → [10]    (독립 공간)
  b → [20]    (독립 공간)
```

### 참조 타입 — 주소 복사

참조 타입(객체, 배열)은 복사할 때 **메모리 주소**가 복사됩니다.  
두 변수가 같은 객체를 가리키게 되어, 하나를 수정하면 다른 쪽도 바뀝니다.

> **비유:** 구글 독스 문서의 **공유 링크**를 복사하는 것과 같습니다.  
> 링크를 두 사람이 가지고 있어도, 실제 문서는 하나입니다.  
> 한 사람이 문서를 수정하면 다른 사람 화면에도 즉시 반영됩니다.

```js
const objA = { name: 'Kim' };
const objB = objA;   // 주소가 복사됨 → 같은 객체를 가리킴

objB.name = 'Lee';
console.log(objA.name); // 'Lee' — objA도 바뀜!
```

```
메모리:
  objA → [주소 #100] ─┐
                        └→ { name: 'Lee' }  (실제 데이터)
  objB → [주소 #100] ─┘
```

### 얕은 복사 vs 깊은 복사

객체를 완전히 독립적으로 복사하려면 **복사 방식**을 신경 써야 합니다.

**얕은 복사(Shallow Copy):** 1단계 속성만 복사합니다.  
객체 안에 또 다른 객체(중첩 객체)가 있으면, 그 안쪽은 여전히 주소를 공유합니다.

**깊은 복사(Deep Copy):** 중첩 구조까지 통째로 새로 복사합니다. 완전히 독립적입니다.

```js
// 얕은 복사: 1단계 속성만 복사 → 중첩 객체는 여전히 공유
const copy = { ...original };

// 깊은 복사: 중첩 구조까지 완전히 새로 복사 → 완전 독립
const deepCopy = structuredClone(original);            // 권장
const deepCopy2 = JSON.parse(JSON.stringify(original)); // 함수/Date 불가
```

```
얕은 복사:
  copy    → { name: 'Lee'  ← 독립 (1단계 속성)
              address ────────→ { city: 'Seoul' }  ← 공유! (중첩 객체)
            }

깊은 복사:
  deepCopy → { name: 'Lee'
               address → { city: 'Seoul' }  ← 완전히 새 객체
             }
```

### 함수에서 원본 보호 패턴

함수에 객체를 넘길 때, 함수 안에서 그 객체를 수정하면 **원본이 바뀝니다.**  
이를 방지하려면 복사본을 만들어 작업해야 합니다.

```js
// ❌ 원본이 변하는 경우
function addJob(person) {
  person.job = 'Dev'; // 전달받은 주소를 통해 원본 수정
}

// ✅ 원본 보호 패턴
function updateName(person) {
  const newPerson = { ...person }; // 복사본을 만들어 작업
  newPerson.name = 'Lee';
  return newPerson;
}
```

> React 같은 프레임워크에서는 원본 데이터를 직접 수정하지 않는 것이  
> 아주 중요한 원칙입니다. 이 패턴을 익혀두면 나중에 큰 도움이 됩니다.

## 2. Truthy / Falsy + 단축 평가 (`abref.js`)

### Falsy 값 7가지

조건문(`if`)에서 `false`로 취급되는 값들입니다.  
이 7가지 외의 **모든 값은 Truthy**입니다.

```js
false, 0, -0, 0n, "", '', ``, null, undefined, NaN
```

> **⚠️ 주의:** 빈 배열 `[]`과 빈 객체 `{}`는 **Truthy**입니다!  
> "비어있으니까 false겠지"라고 생각하기 쉬운데, 이게 자주 발생하는 실수입니다.
> ```js
> Boolean([]) // true — 빈 배열도 true!
> Boolean({}) // true — 빈 객체도 true!
> Boolean('0') // true — 문자열 '0'도 true! (숫자 0은 false)
> ```

### `||` 단축 평가 — 첫 번째 Truthy 반환

`||`는 왼쪽 값을 먼저 평가해서 **Truthy이면 즉시 반환**, Falsy이면 오른쪽을 반환합니다.

```
왼쪽 평가 → Truthy면 즉시 반환
           Falsy면 → 오른쪽 평가 → 그 값 반환
```

```js
null || 'default'    // 'default'
0    || 'default'    // 'default' ← 0도 Falsy 취급
'hi' || 'default'   // 'hi'      ← Truthy이므로 즉시 반환

// 활용: 기본값 설정
const msg = userMsg || '안녕하세요';
```

### `&&` 단축 평가 — 첫 번째 Falsy 반환

`&&`는 왼쪽이 **Falsy이면 즉시 반환**(오른쪽 실행 안 함), Truthy이면 오른쪽을 반환합니다.

```
왼쪽 평가 → Falsy면 즉시 반환
           Truthy면 → 오른쪽 평가 → 그 값 반환
```

```js
false && '실행'    // false  ← Falsy이므로 즉시 반환 (오른쪽 실행 안 됨)
true  && '실행'    // '실행' ← Truthy이므로 오른쪽 반환

// 활용: 조건부 실행 (if 문 대체)
// "로그인한 경우에만 환영 메시지 출력"
isLoggedIn && console.log('환영합니다!');
```

> `&&`의 단축 평가는 "조건이 맞을 때만 실행"하는 패턴에 자주 사용됩니다.  
> React에서 조건부 렌더링(`{isLoggedIn && <WelcomeMessage />}`)에도 이 원리가 그대로 사용됩니다.

### 형 변환

JavaScript는 타입이 맞지 않는 연산을 만나면 **자동으로 타입을 변환**합니다.  
이것을 **묵시적 형 변환**이라고 합니다. 편리하지만 예상치 못한 결과가 나올 수 있어 주의해야 합니다.

```js
// 묵시적 형 변환 (자동)
1 + '2'     // '12'  ← + 는 문자열 우선 (숫자 1이 문자열로 변환됨)
'5' - 2     // 3     ← - 는 숫자로 변환 (문자열 '5'가 숫자로 변환됨)
0 == false  // true  ← == 는 타입 변환 후 비교 (위험!)

// 명시적 형 변환 (의도적)
Number('42')    // 42
Number(null)    // 0
Number(undefined) // NaN
parseInt('42px')  // 42 — 숫자 부분만 추출 (단위 붙은 CSS 값 처리에 유용)
String(42)      // '42'
Boolean(0)      // false
```

> **NaN (Not a Number):** "숫자가 아님"을 나타내는 특별한 값입니다.  
> `Number(undefined)`, `parseInt('abc')` 처럼 숫자로 변환할 수 없을 때 반환됩니다.  
> 특이하게도 `NaN === NaN`은 `false`입니다. NaN 체크는 `Number.isNaN(value)`를 사용합니다.

### `==` vs `===`

```js
1 == '1'    // true  ← 타입 변환 후 비교 (위험)
1 === '1'   // false ← 타입까지 비교 (안전)

// ✅ 항상 === 을 사용합니다
// 예외: null/undefined 동시 체크
val == null  // null과 undefined 모두 잡는 유일한 == 사용 케이스
```

> **`===` 사용 원칙:** `==`는 타입 변환이 일어나면서 예상치 못한 결과를 만듭니다.  
> `"" == false` → `true`, `0 == "0"` → `true` 같은 결과가 나옵니다.  
> 실수를 방지하기 위해 **항상 `===`(엄격 비교)를 사용**합니다.

## 3. Nullish 병합 `??` + 옵셔널 체이닝 `?.` (`logic_null.js`)

### `||` vs `??` 핵심 차이

`||`와 `??`는 둘 다 "기본값을 설정하는" 연산자처럼 보이지만 동작이 다릅니다.

| 연산자 | 기본값 사용 조건 | 0, "" 처리 |
|-------|---------------|-----------|
| `\|\|` | Falsy 값 전체 | 기본값으로 대체 |
| `??` | `null` 또는 `undefined` 만 | 유효한 값으로 유지 |

```js
0  || '기본값'   // '기본값' ← 0을 Falsy로 취급 → 실수 가능!
0  ?? '기본값'   // 0        ← 0은 null/undefined 아님, 유효한 값

'' || '기본값'   // '기본값' ← 빈 문자열을 Falsy로 취급 → 실수 가능!
'' ?? '기본값'   // ''       ← 빈 문자열도 유효한 값
```

> **언제 `??`를 써야 하나?**  
> 사용자가 의도적으로 `0`을 입력했거나, 빈 문자열(`""`)을 입력했을 때 이를 유효한 값으로 유지해야 한다면 `??`를 사용합니다.  
> 예: "몇 개 주문하시겠어요?" → 사용자가 0개 입력 → `||`면 기본값으로 바뀌지만 `??`면 0이 유지됩니다.

### `??` 우선순위 체인

```js
// 사용자 설정 → 저장된 설정 → 기본값 순서로 찾아가는 패턴
const theme = userSetting ?? savedSetting ?? 'light';
// userSetting이 있으면 그것을 사용
// userSetting이 null/undefined이면 savedSetting을 확인
// savedSetting도 null/undefined이면 'light' 사용
```

### 옵셔널 체이닝 `?.`

`null`이나 `undefined`에 `.속성`으로 접근하면 TypeError가 발생합니다.  
이것은 실무에서 매우 자주 만나는 오류입니다.  
`?.`는 이 오류를 방지해 주는 안전 장치입니다.

> **비유:** 주소를 따라 찾아가는데, 중간에 건물이 없을 수 있습니다.  
> `?.`는 "건물이 있으면 들어가고, 없으면 그냥 undefined를 반환해줘"라는 의미입니다.  
> 기존에는 건물이 있는지 확인하고 → 들어가고 → 층이 있는지 확인하고 → 호실을 찾는  
> 복잡한 if 문이 필요했지만, `?.`로 한 줄로 표현할 수 있습니다.

```js
const user = null;

// ❌ 에러 발생
user.name               // TypeError!

// ✅ ?. 사용 — null/undefined면 undefined 반환 (에러 없음)
user?.name              // undefined
user?.address?.city     // undefined (중첩도 가능)
user?.greet?.()         // undefined (메서드 호출도 가능)
arr?.[0]                // undefined (배열 접근도 가능)
```

### `?.` + `??` 조합 패턴

실무에서 가장 자주 사용하는 조합입니다.  
"안전하게 접근 + 없으면 기본값"을 한 줄로 표현합니다.

```js
// 가장 많이 쓰는 실무 패턴: 안전하게 접근 + 기본값
const city = user?.address?.city ?? '도시 정보 없음';
```

```
흐름:
  user가 null → ?. 로 undefined 반환 → ?? 로 '도시 정보 없음' 반환
  user가 있음 → address 접근 → city 없음 → ?? 로 '도시 정보 없음' 반환
  user가 있음 → address 있음 → city = 'Seoul' → ?? 불필요, 'Seoul' 반환
```

### 논리 할당 연산자 (ES2021)

"없을 때만 할당"을 더 짧게 쓸 수 있는 문법입니다.

```js
a ??= '기본값'  // a가 null/undefined 일 때만 할당
b ||= '기본값'  // b가 Falsy 일 때만 할당
c &&= '새값'    // c가 Truthy 일 때만 할당
```

## 핵심 요약

```
원시 타입 복사 → 독립적 (값 복사)
참조 타입 복사 → 연결됨 (주소 복사) → 원본 보호 시 {...obj} 또는 structuredClone()

기본값 설정:
  0, ""도 유효한 값이면 → ??  사용
  0, ""도 기본값으로 대체해도 되면 → || 사용

null/undefined 안전 접근 → ?.  (옵셔널 체이닝)
비교 연산자 → 항상 === 사용 (== 는 사용 금지)
```
